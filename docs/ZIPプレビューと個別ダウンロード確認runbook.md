# ZIPプレビューと個別ダウンロード確認 runbook

この文書は、current `main` の ZIP 添付プレビュー画面と ZIP 項目プレビュー画面の見方を整理する runbook です。

`docs/specs/archive-preview.md` には画面責務と制約が整理されていますが、日常運用では「どこで何を見て、どの操作を使うか」を短く辿れる方が使いやすいため、README / docs index から辿れる運用入口としてまとめます。

## 1. 先に見るもの

1. ZIP preview の責務と制約は [archive preview spec](./specs/archive-preview.md)
2. 文書詳細画面全体の見方は [版詳細プレビュー・差分・添付確認 runbook](./版詳細プレビュー・差分・添付確認runbook.md)
3. 公開モデルと権限境界は [アプリケーション仕様](./アプリケーション仕様.md)

## 2. この runbook が扱う画面

- ZIP 添付を `inline` で開いたときの `ZIPプレビュー` 画面
- ZIP 一覧から `プレビュー` を押したときの `ZIP項目プレビュー` 画面

この 2 画面は役割が違います。

- ZIP 全体の構成、候補、危険な path、有効な操作を見たいときは `ZIPプレビュー`
- 1 つのテキスト項目だけ本文確認したいときは `ZIP項目プレビュー`

## 3. 最初の切り分け順

1. ZIP 全体の件数感と候補数を `ZIP内サマリー` で見る
2. 欲しい path の居場所を `ZIP全体path検索`、`ディレクトリサマリー`、または一覧の表示中検索で絞る
3. 項目ごとに `テキスト確認候補` / `ダウンロード候補` / `操作不可` を見る
4. 必要なら `引き継ぎ用Markdown` で現在の表示条件と件数感だけを read-only に共有する
5. テキストをその場で読みたいときは `プレビュー`
6. 実ファイルが必要なときは `個別ダウンロード`

## 4. `ZIPプレビュー` 画面の見方

### `ZIP全体path検索`

大きい ZIP で先頭表示の外にある path を探したいときは、画面上部の `ZIP全体path検索` を使います。

- ZIP 内の path 全体に対する部分一致検索です
- 一致した項目だけが下の一覧に表示されます
- 検索結果が多い場合は、先頭 N 件だけを表示します
- unsafe path、nested archive、download 可否、text preview 可否の判定は通常一覧と同じです

`ZIP全体path検索` は path だけを探す first slice です。ZIP 内の本文全文検索、directory / safety / candidate の全件 filter、server-side pagination、保存済み filter は current support ではありません。

### `ZIP内サマリー`

- `ファイル数`
- `フォルダ数`
- `テキスト確認候補`
- `ダウンロード候補`
- `合計サイズ`

最初にここを見ると、「中身をざっと確認したい段階か」「候補を絞って探す段階か」を判断しやすくなります。

`ZIP全体path検索` を使っている場合、このサマリーは検索結果として表示されている項目だけを対象にします。

### `引き継ぎ用Markdown`

`引き継ぎ用Markdown` は、現在の ZIP プレビュー条件と件数感を read-only に共有するためのメモです。調査依頼やレビューコメントへ貼る前に、何を含み、何を含まないかを確認します。

含まれるもの:

- 対象ファイル名
- `ZIP全体path検索` の有無。ただし検索語そのものは含めません
- 表示範囲が「検索結果だけ」か「画面に表示されているプレビュー対象だけ」か
- 表示上限と、先頭 N 件までの場合は ZIP 全件保証ではないこと
- 表示中のファイル数 / フォルダ数、要注意 path 件数、テキスト確認候補数、ダウンロード候補数
- 初期 filter 状態と、この runbook への導線

含めないもの:

- ZIP entry 本文
- binary content
- unsafe path の raw 再利用
- secret-like value
- path 検索語の raw value
- 保存済み report、JSON API、CSV export、通知、assignment、ack workflow

candidate 0 件は正常保証ではなく、この表示条件で該当候補がない状態として扱います。path 検索時や大きい ZIP では、表示中または検索結果として見えている範囲の要約であり、ZIP 全件を確認した証跡ではありません。

### `ディレクトリサマリー`

- `ディレクトリ`
- `ファイル数`
- `フォルダ数`
- `直下ファイル合計サイズ`
- `パスをコピー`

path をそのまま人に渡したい、または下の一覧を directory 単位で絞りたいときの入口です。

`ZIP全体path検索` を使っている場合、このディレクトリサマリーは検索結果として表示されている項目だけを対象にします。

### `ZIP内ファイル一覧`

一覧では少なくとも次を見ます。

- `種別`: `ファイル` か `フォルダ`
- `パス`
- `サイズ`
- `安全性`: `安全` または `要注意パス`
- `プレビュー候補`
- `ダウンロード候補`
- `操作`

`要注意パス` は、その項目をそのまま preview / download に使わない方がよいものです。安全な path でないため、runbook 上も再利用前提にしません。

## 5. フィルタと並び替えの使い分け

一覧上部では次の条件を使えます。

- `表示中検索`: 下に表示されている path だけを部分一致で探す
- `ディレクトリ`: directory summary に出ている path 単位で絞る
- `種別`: `ファイル` / `フォルダ`
- `安全性`: `安全` / `要注意パス`
- `候補`: `テキスト確認候補` / `ダウンロード候補` / `操作不可`
- `条件をリセット`: まとめて解除する

並び替えは `種別` `パス` `サイズ` の見出しボタンから切り替えます。

迷ったときの基本:

- path が分かっていて、先頭表示外も探したい: `ZIP全体path検索`
- 表示中の項目だけを素早く絞りたい: `表示中検索`
- 特定ディレクトリだけ見たい: `ディレクトリ`
- 読めるテキストだけ先に洗いたい: `候補 = テキスト確認候補`
- 個別に持ち出せるものだけ見たい: `候補 = ダウンロード候補`
- 危険 path や操作不可理由を確認したい: `安全性` と `候補 = 操作不可`

## 6. `プレビュー` と `個別ダウンロード` の違い

### `プレビュー`

- `テキスト確認候補` の項目にだけ出ます
- 遷移先は `ZIP項目プレビュー` 画面です
- 小さいテキスト項目だけを対象にします
- 一覧から開いた場合、`ZIP項目プレビュー` の `ZIP内ファイル一覧へ戻る` で元の ZIP 一覧の対象行へ戻れます
- 画面内では `検索`、`一致行のみ表示`、`表示中の行をコピー`、`リセット` を使えます

「内容を少し読めれば十分」「差分確認前に本文だけ見たい」という場面に向いています。

`ZIP内ファイル一覧へ戻る` は、一覧の対象行へ戻るための内部 anchor です。`ZIP全体path検索` から開いた場合は、その検索語も戻り先に反映されます。表示中検索、ディレクトリ filter、種別 filter、安全性 filter、候補 filter、並び替えの状態を保存する機能ではありません。追加の絞り込み文脈を残したい場合は、戻ったあとに必要な条件を再度確認します。

### `個別ダウンロード`

- `ダウンロード候補` の項目にだけ出ます
- ZIP 全体ではなく、その項目だけを attachment として取得します
- unsafe path や nested archive 相当の項目では使えません
- `ZIP項目プレビュー` 画面でも、同じ項目がダウンロード候補なら `個別ダウンロード` を直接使えます

「元ファイルそのものが必要」「テキスト以外の項目を取得したい」という場面に向いています。

## 7. `ZIP項目プレビュー` 画面の見方

`ZIP項目プレビュー` では次を確認できます。

- `項目パス`
- `ファイル名`
- `コンテンツタイプ`
- `サイズ`
- 本文行

画面上部の action は次のように使います。

- `ZIP内ファイル一覧へ戻る`: 元の ZIP 一覧を開き、一覧から渡された対象行 anchor と、`ZIP全体path検索` の検索語があればその条件へ戻る
- `個別ダウンロード`: 同じ項目が download candidate の場合だけ表示され、本文確認から実ファイル取得へ切り替える

本文表示では、一覧画面より細かく中身を読むための補助だけを使います。

- `検索`: 項目内の文字列を探す
- `一致行のみ表示`: hit 行だけを見る
- `表示中の行をコピー`: 抜粋共有に使う
- `リセット`: 絞り込み解除

大きい項目では `先頭 N 行を表示しています` が出ることがあります。この場合、全量確認が必要なら `ZIP内ファイル一覧へ戻る` で一覧へ戻って、対象行または path を確認してから `個別ダウンロード` に切り替えます。

## 8. `プレビューできません` / `操作不可` の見方

項目によっては preview や download ができません。current 実装では、少なくとも次のような理由がありえます。

- unsafe path
- directory entry
- text preview 候補ではない binary 項目
- nested archive 相当
- サイズ上限超過

この runbook では「操作できない項目を無理に通す」運用は足しません。理由表示を見て、ZIP 全体ダウンロードや別確認手段へ切り替えます。

## 9. 大きい ZIP のときの注意

`先頭 N 件を表示しています` が出るときは、画面に出ている件数だけを対象に次も計算されています。

- `ZIP内サマリー`
- `ディレクトリサマリー`
- `表示中検索`
- 各種フィルタ
- 並び替え
- コピー操作
- `引き継ぎ用Markdown`

先頭表示外の path を探したいときだけ、`ZIP全体path検索` を使います。検索後のサマリー、ディレクトリ、表示中検索、各種フィルタ、並び替え、コピー操作、`引き継ぎ用Markdown` は、検索結果として表示されている項目だけが対象です。

そのため、全項目を前提に判断したいときや本文まで横断検索したいときは ZIP 全体をダウンロードして確認します。

## 10. 迷ったときの切り分け

- ZIP 全体の規模を知りたい: `ZIP内サマリー`
- どのディレクトリに何があるか見たい: `ディレクトリサマリー`
- 先頭表示外の path を探したい: `ZIP全体path検索`
- 表示中の項目だけを絞りたい: `表示中検索`
- 読める text だけ先に見たい: `候補 = テキスト確認候補`
- 取得できる項目だけ見たい: `候補 = ダウンロード候補`
- 現在の表示条件と件数感だけを共有したい: `引き継ぎ用Markdown`
- 中身をその場で読みたい: `プレビュー`
- preview 後に周辺項目へ戻りたい: `ZIP内ファイル一覧へ戻る`
- preview 中の同じ項目を取得したい: `個別ダウンロード`
- 実ファイルが必要: `個別ダウンロード`
- 画面に出ていない項目まで確認したい: ZIP 全体をダウンロードする
