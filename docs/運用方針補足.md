# 運用方針補足

この文書は、現行実装と運用判断に合わせて、`docs/アプリケーション仕様.md` の補足・訂正として扱う。

## `latest_version` の決定基準

`latest_version` は、原則として `published` な `DocumentVersion` のうち、最も新しく作成されたものを指す。

`version_label` は表示・識別用の値であり、最新版判定の正本にはしない。利用者や投入元によって `version_label` の表記ゆれが大きいため、semantic version として機械的に比較できることを前提にしない。

したがって、`version_label` が `1.10.0` / `v2` / `current` / `提出済` のように混在していても、標準の最新版判定は `created_at` ベースで考える。

`draft` / `archived` な `DocumentVersion` は、引き続き `latest_version` の候補にしない。

## Docusaurus asset / metadata の露出許容範囲

Docusaurus build 成果物の配信では、重要な権限境界は「権限のない利用者がドキュメント本文・添付ファイル・対象 HTML 本体にアクセスできないこと」とする。

画面上では権限外文書のタイトルや導線を出さない。ただし、Docusaurus の静的 asset、client bundle、developer tools などを通じて、slug・一部 metadata・タイトル相当の情報が技術的に見える可能性は許容する。

この方針では、CSS / JS / image などの project site asset は Document 単位では分離しない。metadata レベルまで厳密に秘匿する必要が出た場合は、利用者権限ごとの site build 分離や asset 分離を別途検討する。
