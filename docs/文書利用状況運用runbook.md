# 文書利用状況運用runbook

この runbook は、管理画面の `文書利用状況` を日常運用で見返すときの入口をまとめる。

新しい KPI や集計ルールはここでは定義しない。current 実装を前提に、案件単位の集計をどう読み、必要に応じてどこへ戻るかを整理する。

## 先に見るもの

1. 文書公開モデルや権限前提を確認したいときは [アプリケーション仕様](./アプリケーション仕様.md)
2. アクセス証跡の見方を先に確認したいときは [監査ログ運用runbook](./監査ログ運用runbook.md)
3. 管理画面の位置関係は `app/views/admin/_nav.html.slim` の `文書利用状況` を起点に確認する

## 画面の役割

`admin/document_usage_reports` は internal admin だけが使える案件単位の集計画面で、文書ごとの利用有無と総量をざっと見返すために使う。

current 実装の前提:

- 最初に `project_id` で案件を 1 つ選んでから集計する
- 集計対象は選んだ案件に属する文書で、文書一覧は title 順で並ぶ
- `利用あり` は `閲覧` `ダウンロード` `既読確認` のどれかが 1 回以上ある文書を指す
- `最終アクセス` は AccessLog の `accessed_at` を元にした最新時刻で、既読確認時刻はここには入らない
- current UI に期間指定はなく、案件全体の累積集計を出す

## 集計サマリの見方

### 1. 文書数 / 利用あり / 未利用

- `文書数` は案件配下の文書総数
- `利用あり` は少なくとも 1 つの利用実績がある文書数
- `未利用` は閲覧・ダウンロード・既読確認が 1 度もない文書数

### 2. 閲覧 / ダウンロード / 既読確認

- `閲覧` と `ダウンロード` は AccessLog 由来の累積件数
- `既読確認` は ReadConfirmation の累積件数
- 文書ごとの傾向を見る前に、案件全体でどの利用が多いかをざっくり把握するのに向く

## 一覧列の見方

- `文書名`: project/document detail への戻り口
- `カテゴリ`: current document category の表示
- `種別`: current document kind の表示
- `公開範囲`: visibility policy の表示
- `利用`: `あり` / `なし` で利用実績の有無を示す
- `閲覧`: AccessLog 上の view 件数
- `ダウンロード`: AccessLog 上の download 件数
- `既読確認`: ReadConfirmation 件数
- `最終アクセス`: AccessLog 上で最後に記録された時刻。アクセスが無ければ `-`

## 日常確認ポイント

- 案件内で、公開中の文書に未利用が偏っていないか
- 閲覧だけ多い文書と、ダウンロードまで進む文書の差が大きすぎないか
- 既読確認が必要な運用で、確認数が極端に少ない文書がないか
- 気になる文書があれば文書 detail へ戻り、公開状態や添付構成を確認する必要があるか

## 変更時の注意

- current UI は案件を 1 つずつ見る画面で、会社横断や利用者横断の比較画面ではない
- `最終アクセス` はアクセスログ由来なので、既読確認だけが行われた文書では空欄のままになりうる
- 集計ロジックや KPI 定義を増やす docs ではなく、いまの画面の読み方だけに留める
- company master admin や external user は使えない画面なので、社外利用者向け runbook と混ぜない

## 戻り先

- 個別のアクセス履歴を追いたいときは [監査ログ運用runbook](./監査ログ運用runbook.md)
- 文書の公開状態や版状態を見直したいときは [文書ライフサイクルと公開](./specs/文書ライフサイクルと公開.md)
- 権限や閲覧可能範囲を見直したいときは [基本モデルと権限](./specs/基本モデルと権限.md)

## 迷ったときの切り分け

- 案件全体で未利用文書や利用偏りを見たい: `文書利用状況` を使う
- 誰がいつ見たかを追いたい: `監査ログ` に戻る
- その文書が見える理由や公開条件を確認したい: 仕様 docs と文書 detail に戻る
- 期間比較や高度な集計をしたい: current 画面の責務外として扱う
