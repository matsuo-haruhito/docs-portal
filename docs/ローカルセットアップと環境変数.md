# ローカルセットアップと環境変数

この文書は、`docs-portal` をローカルで起動するときの最短手順と、`.env.example` を基準にした環境変数の役割をまとめる正本です。

詳細な preview runtime や外部連携の仕様は既存文書へ分け、このページでは「まず何を起動し、どの変数がどこに効くか」を一か所で追える形に絞ります。

## 最短起動手順

最小の開発構成は `app + db` です。`.env.example` をそのまま起点にし、必要なら追加の compose file を有効化します。

```bash
cp .env.example .env
docker compose build app
docker compose run --rm app bin/setup --skip-server
docker compose up app
```

- `bin/setup` は `bundle install` の確認、`bin/rails db:prepare`、`log:clear tmp:clear` をまとめて実行します。
- `docker-compose.yml` の `app` は `db` に依存しているため、`docker compose up app` で `db` も一緒に起動します。
- `config/database.yml` は `DATABASE_NAME` を基底名として読み、development は `<DATABASE_NAME>_development`、test は `<DATABASE_NAME>_test` を使います。

### seed サンプルも確認したい場合

```bash
docker compose run --rm app bin/setup_external_sample_data_links
docker compose run --rm app bin/rails db:seed
```

`bin/setup_external_sample_data_links` は `storage/document_files/external_samples/` を用意するだけなので、外部サンプル文書を置きたいときの入口として使います。

## optional service の有効化

`.env.example` の既定値は `COMPOSE_FILE=docker-compose.yml` です。追加サービスを使うときだけ `COMPOSE_FILE` を増やします。

### Manual Markdown / MDX preview を使う

```env
COMPOSE_FILE=docker-compose.yml:docker-compose.docusaurus.yml
DOCUSAURUS_RENDERER_ENDPOINT=http://docusaurus:3000
```

### Kroki を使って PlantUML / D2 も preview/build する

```env
COMPOSE_FILE=docker-compose.yml:docker-compose.kroki.yml:docker-compose.docusaurus.yml
DOCUSAURUS_RENDERER_ENDPOINT=http://docusaurus:3000
KROKI_ENDPOINT=http://kroki:8000
```

追加 compose file の役割は次のとおりです。

- `docker-compose.docusaurus.yml`: manual upload preview 用の renderer service を追加します。
- `docker-compose.kroki.yml`: Kroki service を追加し、diagram renderer の endpoint を有効化します。

runtime 前提や upload size 制限の詳細は [notes/docusaurus-build-runtime](./notes/docusaurus-build-runtime.md) を参照してください。

## 環境変数一覧

### Rails / DB / 暗号化の基本値

| 変数 | 既定値 | 役割 |
| --- | --- | --- |
| `COMPOSE_FILE` | `docker-compose.yml` | 起動する compose file の組み合わせです。Docusaurus renderer や Kroki を使うときだけ追加します。 |
| `APP_UID` / `APP_GID` | `1000` | 開発コンテナ内の `docker` ユーザーの uid / gid です。ホスト側のファイル権限と合わせたいときに使います。 |
| `DATABASE_HOST` | `db` | Rails が接続する PostgreSQL service 名です。 |
| `DATABASE_PORT` | `5432` | PostgreSQL の接続先 port です。 |
| `DATABASE_USER` / `DATABASE_PASSWORD` | `postgres` / `docs_portal` | PostgreSQL 認証情報です。 |
| `DATABASE_NAME` | `docs_portal` | DB 名の基底です。development / test では suffix 付きで使います。 |
| `RAILS_MASTER_KEY` | `replace_me` | Rails credentials を読むための master key です。 |
| `SECRET_KEY_BASE` | `secret` | session / verifier 用の secret です。 |
| `ACTIVE_RECORD_ENCRYPTION_PRIMARY_KEY` | `replace_me` | 暗号化カラムの primary key です。 |
| `ACTIVE_RECORD_ENCRYPTION_DETERMINISTIC_KEY` | `replace_me` | deterministic encryption 用 key です。 |
| `ACTIVE_RECORD_ENCRYPTION_KEY_DERIVATION_SALT` | `replace_me` | Active Record encryption の salt です。 |
| `ACTIVE_STORAGE_SERVICE` | `local` | Active Storage の保存先 service 名です。 |

### Rails runtime と logging

| 変数 | 既定値 | 役割 |
| --- | --- | --- |
| `RAILS_SERVE_STATIC_FILES` | `true` | Rails が静的ファイルを配信する前提を有効にします。 |
| `RAILS_FORCE_SSL` | 空 | SSL 強制の切り替えです。ローカルでは通常空のまま使います。 |
| `RAILS_LOG_TO_STDOUT` | 空 | stdout logging を使うときに有効化します。 |
| `RAILS_LOG_SHIFT_AGE` | `5` | log rotation 世代数です。 |
| `RAILS_COLORIZE_LOGGING` | `true` | 開発ログの色付けです。 |
| `RAILS_MAX_THREADS` | `5` | Rails / DB 設定が参照する最大 thread 数です。 |

### import / preview / 外部連携

| 変数 | 既定値 | 役割 |
| --- | --- | --- |
| `DOC_IMPORT_TOKEN` | `local-dev-import-token` | internal import API の token です。`storage/imports/` を読む import 経路の認証に使います。 |
| `DOCUSAURUS_RENDERER_ENDPOINT` | `http://docusaurus:3000` | manual Markdown / MDX preview build を受ける renderer service の endpoint です。 |
| `DOCUSAURUS_RENDERER_MAX_UPLOAD_BYTES` | `20971520` | renderer に送る source archive の上限 byte 数です。 |
| `DOCUSAURUS_RENDERER_MAX_OUTPUT_BYTES` | `52428800` | renderer が返す build artifact の上限 byte 数です。 |
| `DOCUSAURUS_RENDERER_BUILD_TIMEOUT_MS` | `60000` | preview build の timeout です。 |
| `KROKI_ENDPOINT` | `http://kroki:8000` | Docusaurus build / preview で diagram renderer を使う endpoint です。 |
| `GOOGLE_DRIVE_OAUTH_CLIENT_ID` | `replace_me` | Google Drive OAuth user 方式と upload preview の client id です。 |
| `GOOGLE_DRIVE_OAUTH_CLIENT_SECRET` | `replace_me` | Google Drive OAuth user 方式と upload preview の client secret です。 |
| `GOOGLE_DRIVE_PREVIEW_FOLDER_ID` | `replace_me` | preview 用に一時 upload する Google Drive folder id です。 |
| `GOOGLE_DRIVE_PREVIEW_UPLOAD_TTL_HOURS` | `168` | 一時 upload preview の有効時間の既定値です。 |

### 公開 port

| 変数 | 既定値 | 役割 |
| --- | --- | --- |
| `PUBLISH_WEB_SERVER_PORT` | `3000` | ローカル Rails app を公開する port です。 |
| `PUBLISH_DATABASE_PORT` | `5432` | PostgreSQL をホストへ公開する port です。 |
| `PUBLISH_DOCUSAURUS_RENDERER_PORT` | `3001` | Docusaurus renderer をホストへ公開する port です。 |

`PUBLISH_KROKI_PORT` は `docker-compose.kroki.yml` 側で `8000` を既定値として参照します。Kroki を有効化したときだけ必要に応じて上書きしてください。

## どの機能で何が必要か

- ローカルで Rails 画面を見るだけなら、まずは `app + db` と Rails / DB / 暗号化まわりの値をそろえます。
- manual Markdown / MDX preview を確認したいときは、Docusaurus renderer 関連の compose file と `DOCUSAURUS_RENDERER_*` を有効化します。
- PlantUML / D2 を含む preview や seed build を確認したいときは、さらに Kroki を有効化します。
- Google Drive 同期や upload preview を確認したいときは、`GOOGLE_DRIVE_*` を設定したうえで [Google Drive外部フォルダ同期](./Google%20Drive外部フォルダ同期.md) の OAuth / refresh token 前提も確認します。
- Office preview の fallback 条件を確認したいときは、[Microsoft Graph接続とOffice preview](./Microsoft%20Graph%E6%8E%A5%E7%B6%9A%E3%81%A8Office%20preview.md) と合わせて見ます。
