# 文書ショートカット運用runbook

この runbook は、`文書ショートカット` 画面を current UI に沿って見返すための運用メモです。

新しいショートカット種別や並び替えルールはここでは定義しません。current 実装を前提に、`お気に入り` `後で読む` `最近見た文書` をどう読み分けるか、文書詳細から追加したあとや一覧で `解除` したあとにどこへ戻るかを整理します。

## 先に見るもの

1. dashboard からの入口と個人導線の役割差は [ダッシュボードと文書ショートカット・確認依頼の使い分け](./ダッシュボードと文書ショートカット・確認依頼の使い分け.md)
2. 権限や利用者種別の前提は [アプリケーション仕様](./アプリケーション仕様.md) と [基本モデルと権限](./specs/基本モデルと権限.md)
3. 文書詳細から `お気に入りに追加` / `後で読むに追加` する current 導線は文書詳細画面を正本に確認する

## 画面の役割

`GET /document_bookmarks` は、current user が自分用に残した文書ショートカットをまとめて見直す画面です。

current 実装の前提:

- `お気に入り` と `後で読む` は `current_user.document_bookmarks` から作る個人用ショートカットです
- `最近見た文書` は bookmark ではなく、閲覧履歴ベースの最近参照一覧です
- どの一覧も current user から見て readable な文書だけを表示します
- `お気に入り` と `後で読む` は新しい追加順で並びます
- `最近見た文書` は `RecentDocumentsQuery` の current result を最大 20 件まで表示します

## 最初の見分け方

1. 継続的に開く文書を置いておきたいなら `お気に入り` を見る
2. 今すぐではないが近いうちに確認したい文書を溜めているなら `後で読む` を見る
3. 直前に見ていた文書へ戻りたいなら `最近見た文書` を見る
4. どの案件の文書かを先に確かめたいときは、各行の文書名の下に出る案件名を読む

## `お気に入り` の見方

`お気に入り` は、継続的に参照する文書を置いておく一覧です。

各行で見えるもの:

- 文書タイトル: 文書詳細へのリンク
- 案件名: どの案件配下の文書かを見分ける補助情報
- badge `よく開く文書`: current row が favorite であることを示す表示
- `解除`: この favorite bookmark を削除するボタン

0 件のときは `お気に入りはまだありません。` を表示します。

## `後で読む` の見方

`後で読む` は、今は開かないが確認待ちとして残しておきたい文書を置く一覧です。

各行で見えるもの:

- 文書タイトル: 文書詳細へのリンク
- 案件名: どの案件配下の文書かを見分ける補助情報
- badge `あとで確認`: current row が read_later bookmark であることを示す表示
- `解除`: この read_later bookmark を削除するボタン

0 件のときは `後で読む文書はまだありません。` を表示します。

## `最近見た文書` の見方

`最近見た文書` は、bookmark と違って手動で積み上げる一覧ではなく、直近に閲覧した文書へ戻るための一覧です。

各行で見えるもの:

- 文書タイトル: 文書詳細へのリンク
- 案件名: どの案件配下の文書かを見分ける補助情報
- badge `最近見た文書`: current row が recent history 由来であることを示す表示

current UI では `最近見た文書` に `解除` ボタンはありません。0 件のときは `最近見た文書はまだありません。` を表示します。

## 追加と解除の current flow

文書詳細の `管理・補助操作` には、次の導線があります。

- `お気に入りに追加`
- `後で読むに追加`
- `文書ショートカット一覧`

`お気に入りに追加` と `後で読むに追加` は、どちらも文書詳細から `document_bookmarks#create` を呼び、current 実装では `redirect_to_back` で元の画面へ戻ります。通常は追加元の文書詳細へ戻り、notice だけが出ます。

`解除` は `document_bookmarks#destroy` を呼び、こちらも `redirect_to_back` で元の画面へ戻ります。`文書ショートカット` 画面で押したときは、この一覧へ戻って再描画される前提です。Referer が取れない場合の fallback は `root_path` です。

画面下部の `案件一覧へ戻る` は `GET /projects` への固定リンクです。bookmark を整理し終えたあと、案件単位の導線へ戻したいときに使います。

## 日常確認ポイント

- その文書が継続参照用なのか、一時保留なのかを `お気に入り` と `後で読む` で分けられているか
- 文書名だけでは迷う場合に、案件名を見て正しい案件配下の文書か確認できるか
- `最近見た文書` に出ているものが、直前に見直したい導線として足りているか
- 不要になった favorite / read_later を `解除` しても困らないか
- bookmark ではなく文書本文や案件配下の一覧へ戻りたい場面では、文書タイトルリンクや `案件一覧へ戻る` を使い分けられているか

## 変更時の注意

- current UI は個人用ショートカットの見直しを扱う画面で、共有リストや優先度ルールを定義する画面ではありません
- `お気に入り` と `後で読む` の違いは current bookmark_type に基づくため、新しい種別を前提にした説明はここへ足しません
- `最近見た文書` は history ベースなので、bookmark と同じ `解除` や並び替え前提で説明しません
- readable でなくなった文書は一覧に出ないため、表示されない文書をこの runbook で復旧手順扱いにはしません

## 迷ったときの切り分け

- 継続して読む文書を残したい: `お気に入り`
- 後で確認する文書を一時的に積みたい: `後で読む`
- さっき見ていた文書へ戻りたい: `最近見た文書`
- 文書を追加したあと元の文書詳細へ戻りたい: 文書詳細の `お気に入りに追加` / `後で読むに追加` を使う
- 一覧から不要な bookmark を外したい: `解除`
- bookmark 整理のあと案件単位の導線へ戻したい: `案件一覧へ戻る`
