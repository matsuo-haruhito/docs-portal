# 文書ショートカット運用runbook

この runbook は、`文書ショートカット` 画面を current UI に沿って見返すための運用メモです。

新しいショートカット種別や並び替えルールはここでは定義しません。current 実装を前提に、`お気に入り` `後で読む` `最近見た文書` をどう読み分けるか、保存済みショートカットをどう絞り込むか、文書詳細から追加したあとや一覧で `解除` したあとにどこへ戻るかを整理します。

## 先に見るもの

1. dashboard からの入口と個人導線の役割差は [ダッシュボードと文書ショートカット・確認依頼の使い分け](./ダッシュボードと文書ショートカット・確認依頼の使い分け.md)
2. 権限や利用者種別の前提は [アプリケーション仕様](./アプリケーション仕様.md) と [基本モデルと権限](./specs/基本モデルと権限.md)
3. 文書詳細から `お気に入りに追加` / `後で読むに追加` する current 導線は文書詳細画面を正本に確認する

## 画面の役割

`GET /document_bookmarks` は、current user が自分用に残した文書ショートカットをまとめて見直す画面です。

current 実装の前提:

- `お気に入り` と `後で読む` は `current_user.document_bookmarks` から作る個人用ショートカットです
- `お気に入り` と `後で読む` は別の `bookmark_type` として保存されます
- 同じ文書でも種別が違う場合は別 bookmark として扱われるため、current UI では両方の section に同じ文書が出ることがあります
- `保存済みショートカットの絞り込み` は、明示的に保存した `お気に入り` と `後で読む` だけを対象にします
- 保存済み側の form 直下には `対象: お気に入り・後で読む` の badge が出て、最近見た文書は下の検索で絞り込むことを明示します
- `お気に入り` と `後で読む` はそれぞれ独立した page として最大 20 件ずつ表示する。summary label、表示範囲、条件一致件数、`前へ` / `次へ` は section ごとに読む
- 保存済み bookmark の page link と行 action は、`project_code` / `bookmark_q` / `recent_q` と、もう片方の保存済み section の page を維持する。片方の page 移動や `解除` / `お気に入りへ移す` が recent search や相手 section の page を消す前提では読まない
- `最近見た文書` は bookmark ではなく、閲覧履歴ベースの最近参照一覧です
- 最近見た文書側には `対象: 最近見た文書（表示中最大 20 件）` の badge が出て、保存済みショートカットの条件は維持されることを明示します
- どの一覧も current user から見て readable な文書だけを表示します
- `お気に入り` と `後で読む` は新しい追加順で並びます
- `最近見た文書` は `RecentDocumentsQuery` の current result を最大 20 件まで表示します

## 最初の見分け方

1. 継続的に開く文書を置いておきたいなら `お気に入り` を見る
2. 今すぐではないが近いうちに確認したい文書を溜めているなら `後で読む` を見る
3. 保存済みショートカットが増えてきたら、上部の `保存済みショートカットの絞り込み` で案件や検索語を指定する
4. 直前に見ていた文書へ戻りたいなら `最近見た文書` を見る
5. どの案件の文書かを先に確かめたいときは、各行の文書名の下に出る案件名を読む

## 保存済みショートカットの絞り込み

画面上部の `保存済みショートカットの絞り込み` は、`お気に入り` と `後で読む` の両方を同じ条件で絞り込むための form です。form 直下の `対象: お気に入り・後で読む` badge は、この条件が保存済みショートカットだけに効くことを示す cue です。

絞り込み項目:

- `案件`: current user の readable な保存済みショートカットに紐づく案件から選ぶ。候補は案件名を主表示にし、案件コードがある場合は `案件名（案件コード）` として併記されます
- `保存済みショートカットを検索`: 文書名、案件名、案件コードを部分一致で探す

検索語は `DocumentBookmarksController::BOOKMARK_QUERY_MAX_LENGTH` に合わせて最大 100 文字に丸められます。長い検索語や前後の空白は、controller 側で strip / slice された値として扱われます。

絞り込み中の読み方:

- 案件と検索語を両方指定すると、その案件内で文書名・案件名・案件コードに一致する `お気に入り` と `後で読む` だけを表示します
- 案件だけを指定すると、その案件の `お気に入り` と `後で読む` だけを表示します
- 検索語だけを指定すると、全案件の保存済みショートカットから一致するものを表示します
- 指定した案件コードが候補にない場合は、保存済みショートカットが見つからない状態として読みます
- `保存済み条件を解除` は保存済みショートカットの絞り込みだけを外し、`recent_q` があれば最近見た文書側の検索条件は維持します
- 保存済み条件が適用中で `お気に入り` または `後で読む` が 0 件になった場合、各 section の 0 件 message の近くにも `保存済み条件を解除` link が出ます。この link は押した section の anchor に戻り、`project_code` / `bookmark_q` と保存済み section の page params を外し、`recent_q` だけを維持します

`保存済みショートカットの絞り込み` は `最近見た文書` には効きません。最近見た文書を探す場合は、`最近見た文書` section 内の `最近見た文書を検索` を使います。逆に `recent_q` は `お気に入り` / `後で読む` の検索条件ではありません。

## `お気に入り` の見方

`お気に入り` は、継続的に参照する文書を置いておく一覧です。

各行で見えるもの:

- 文書タイトル: 文書詳細へのリンク
- 案件名: どの案件配下の文書かを見分ける補助情報
- badge `よく開く文書`: current row が favorite であることを示す表示
- `解除`: この favorite bookmark を削除するボタン

表示範囲と page:

- section 見出しの summary label は、現在の page に表示している件数と条件一致総件数を表す
- 表示範囲 badge がある場合は、条件一致件数のうち何件目から何件目を見ているかを示す。最大 20 件ずつ表示するため、20 件を超えると page 移動で続きを見る
- 複数 page があるときだけ `前へ` / `次へ` が出る。page link は保存済み filter、recent search、`後で読む` 側の現在 page を維持する

0 件の読み方:

- 絞り込みなしで 0 件なら、文書画面でお気に入りに追加すると表示される初期状態です
- 保存済みショートカット検索中の 0 件なら、検索語に一致するお気に入りがない状態です
- 案件 filter 中の 0 件なら、その案件ではお気に入りが見つからない状態です
- 保存済み条件が適用中なら、この section 近くの `保存済み条件を解除` から保存済み条件だけを外せます。`recent_q` は維持されるため、最近見た文書の検索文脈はそのまま残ります
- 絞り込みなしの初期 0 件では、解除する保存済み条件がないため section-local の `保存済み条件を解除` link は出ません

## `後で読む` の見方

`後で読む` は、今は開かないが確認待ちとして残しておきたい文書を置く一覧です。

各行で見えるもの:

- 文書タイトル: 文書詳細へのリンク
- 案件名: どの案件配下の文書かを見分ける補助情報
- badge `あとで確認`: current row が read_later bookmark であることを示す表示
- `お気に入りへ移す`: この read_later bookmark の文書を favorite として残し、元の read_later bookmark を削除するボタン
- `解除`: この read_later bookmark を削除するボタン

表示範囲と page:

- section 見出しの summary label は、現在の page に表示している件数と条件一致総件数を表す
- 表示範囲 badge がある場合は、条件一致件数のうち何件目から何件目を見ているかを示す。最大 20 件ずつ表示するため、20 件を超えると page 移動で続きを見る
- 複数 page があるときだけ `前へ` / `次へ` が出る。page link は保存済み filter、recent search、`お気に入り` 側の現在 page を維持する
- `お気に入りへ移す` と `解除` は現在の保存済み filter、recent search、`お気に入り` / `後で読む` の page を持ち回る。操作後も同じ文脈で一覧を再表示する導線として読む

0 件の読み方:

- 絞り込みなしで 0 件なら、文書画面で後で読むに追加すると表示される初期状態です
- 保存済みショートカット検索中の 0 件なら、検索語に一致する後で読む文書がない状態です
- 案件 filter 中の 0 件なら、その案件では後で読む文書が見つからない状態です
- 保存済み条件が適用中なら、この section 近くの `保存済み条件を解除` から保存済み条件だけを外せます。`recent_q` は維持されるため、最近見た文書の検索文脈はそのまま残ります
- 絞り込みなしの初期 0 件では、解除する保存済み条件がないため section-local の `保存済み条件を解除` link は出ません

## `最近見た文書` の見方

`最近見た文書` は、bookmark と違って手動で積み上げる一覧ではなく、直近に閲覧した文書へ戻るための一覧です。

各行で見えるもの:

- 文書タイトル: 文書詳細へのリンク
- 案件名: どの案件配下の文書かを見分ける補助情報
- badge `最近見た文書`: current row が recent history 由来であることを示す表示

`最近見た文書` section の `対象: 最近見た文書（表示中最大 20 件）` badge は、この検索が保存済みショートカットではなく recent history 側だけに効くことを示す cue です。続く `保存済みショートカットの条件はそのまま維持されます。` は、保存済み側の `project_code` / `bookmark_q` を hidden field で持ち回ることを示します。

`最近見た文書を検索` は、最近見た文書 section の中だけを文書名・案件名で絞り込みます。検索対象は `RecentDocumentsQuery` が表示する最大 20 件内です。この form は保存済みショートカットの `project_code` / `bookmark_q` を hidden field で維持しますが、最近見た文書の検索語そのものは `recent_q` として別に扱います。`recent_q` も `BOOKMARK_QUERY_MAX_LENGTH` に合わせて strip 後の最大 100 文字として扱われ、入力欄にも同じ上限が設定されています。`最近見た条件をクリア` は `recent_q` だけを外し、保存済みショートカット側の条件は維持します。

current UI では `最近見た文書` に `解除` ボタンはありません。閲覧履歴がない 0 件のときは `文書を開くと、最近見た文書としてここに表示されます。` を表示します。閲覧履歴がある状態で検索に一致しない場合は、検索語を示しながら `最近表示された最大 20 件内` に一致する文書がないことを表示します。

## 追加と解除の current flow

文書詳細の `管理・補助操作` には、次の導線があります。

- `お気に入りに追加`
- `後で読むに追加`
- `文書ショートカット一覧`

`お気に入りに追加` と `後で読むに追加` は、どちらも文書詳細から `document_bookmarks#create` を呼び、current 実装では `redirect_to_back` で元の画面へ戻ります。通常は追加元の文書詳細へ戻り、notice だけが出ます。

同じ文書を `後で読む` から `お気に入り` へ移したい場合、`後で読む` section の `お気に入りへ移す` を使います。current 実装では `document_bookmarks#move_to_favorite` が read_later bookmark を読み取り、同じ文書の favorite bookmark を `find_or_create_by!` で用意してから、元の read_later bookmark を削除します。すでに同じ文書の favorite bookmark がある場合はそれを再利用し、read_later 側だけが一覧から外れます。処理後は `redirect_to_back` で元の画面へ戻り、notice `お気に入りへ移しました。` が出ます。

`解除` は `document_bookmarks#destroy` を呼び、こちらも `redirect_to_back` で元の画面へ戻ります。`文書ショートカット` 画面で押したときは、保存済み filter、recent search、`お気に入り` / `後で読む` の page を持った一覧へ戻って再描画される前提です。Referer が取れない場合の fallback は `root_path` です。

画面下部の `案件一覧へ戻る` は `GET /projects` への固定リンクです。bookmark を整理し終えたあと、案件単位の導線へ戻したいときに使います。

## 日常確認ポイント

- その文書が継続参照用なのか、一時保留なのかを `お気に入り` と `後で読む` で分けられているか
- 同じ文書が両方の section にある場合、継続参照と一時保留の両方として残したいのか、`後で読む` の `お気に入りへ移す` または片方の `解除` で整理すれば足りるのかを確認する
- 保存済みショートカットが増えたとき、案件 filter や文書名・案件名・案件コード検索で目的の文書へ絞り込めるか
- 案件 filter の候補で案件名が似ている場合、併記された案件コードも見て正しい案件か確認できるか
- 保存済みショートカットが 20 件を超えるとき、`お気に入り` と `後で読む` の page、表示範囲、条件一致件数をそれぞれ見て、片方の page 移動がもう片方の表示文脈を壊していないか
- 保存済みショートカットの検索結果が 0 件のとき、未登録なのか、案件や検索語で絞り込まれているだけなのかを見分けられるか
- 保存済み条件で 0 件になった section では、近くの `保存済み条件を解除` link が保存済み条件だけを外し、最近見た文書の検索条件を残す導線だと読めるか
- 文書名だけでは迷う場合に、案件名を見て正しい案件配下の文書か確認できるか
- `最近見た文書` に出ているものが、直前に見直したい導線として足りているか
- 最近見た文書検索で見つからないとき、表示中の最大 20 件内にないだけなのか、検索語が違うのかを切り分けられるか
- 不要になった favorite / read_later を `解除` しても困らないか
- bookmark ではなく文書本文や案件配下の一覧へ戻りたい場面では、文書タイトルリンクや `案件一覧へ戻る` を使い分けられているか

## 変更時の注意

- current UI は個人用ショートカットの見直しを扱う画面で、共有リストや優先度ルールを定義する画面ではありません
- `お気に入り` と `後で読む` の違いは current bookmark_type に基づくため、新しい種別を前提にした説明はここへ足しません
- `お気に入り` と `後で読む` は別種別として併存できますが、`後で読む` の `お気に入りへ移す` は favorite bookmark を用意したうえで元の read_later bookmark を削除する移動導線です
- `お気に入りへ移す` は `後で読む` section の行 action であり、`お気に入り` section や `最近見た文書` に同じ移動 button がある前提では説明しません
- `保存済みショートカットの絞り込み` は `お気に入り` と `後で読む` の表示補助であり、bookmark_type、readable scope、文書詳細へのリンク先、解除処理を変更しません
- 保存済み bookmark は `お気に入り` / `後で読む` それぞれ最大 20 件ずつの page 表示として読む。page、表示範囲、条件一致総件数は保存済み bookmark の一覧表示の境界であり、recent history の取得件数、bookmark type、readable scope、filter 条件そのものを変えるものではありません
- page link や `解除` / `お気に入りへ移す` は、保存済み filter、recent search、相手 section の page を維持する文脈保持のための param を持つ。page が変わったことを新しい検索条件や保存済み条件解除として書かない
- section-local の `保存済み条件を解除` は、保存済み filter で 0 件になった `お気に入り` / `後で読む` から同じ section へ戻るための補助 link です。`recent_q` 以外の保存済み条件と保存済み page params を外す導線であり、recent search の解除や bookmark の削除として書かない
- `bookmark_q` は保存済みショートカット検索、`recent_q` は最近見た文書検索として分けます。片方を変更しても、もう片方の検索仕様が変わったとは書きません
- `最近見た文書` は history ベースなので、bookmark と同じ `解除` や並び替え前提で説明しません
- `最近見た文書を検索` は表示中の最大 20 件内を絞り込む導線です。閲覧履歴全体を横断検索する前提では説明しません
- readable でなくなった文書は一覧に出ないため、表示されない文書をこの runbook で復旧手順扱いにはしません

## 迷ったときの切り分け

- 継続して読む文書を残したい: `お気に入り`
- 後で確認する文書を一時的に積みたい: `後で読む`
- お気に入り / 後で読むを案件や検索語で絞りたい: `保存済みショートカットの絞り込み`
- 保存済み条件で 0 件になった `お気に入り` / `後で読む` から絞り込み前へ戻りたい: section 近くの `保存済み条件を解除`
- 保存済みショートカットが20件を超える: `お気に入り` / `後で読む` それぞれの `前へ` / `次へ` と表示範囲を見て、必要な page へ移動する
- さっき見ていた文書へ戻りたい: `最近見た文書`
- 最近見た文書だけを文書名・案件名で探したい: `最近見た文書を検索`
- `後で読む` を `お気に入り` 扱いに変えたい: `後で読む` の `お気に入りへ移す`
- 文書を追加したあと元の文書詳細へ戻りたい: 文書詳細の `お気に入りに追加` / `後で読むに追加` を使う
- 一覧から不要な bookmark を外したい: `解除`
- bookmark 整理のあと案件単位の導線へ戻りたい: `案件一覧へ戻る`
