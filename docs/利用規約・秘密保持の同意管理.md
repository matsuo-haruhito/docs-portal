# 利用規約・秘密保持の同意管理

この文書は issue `#130` に対応する、利用規約・秘密保持・注意事項への同意管理の初期実装範囲を記録する。

## 目的

- Project やダウンロード操作の前に、利用者へ必要な規約・NDA・注意事項を提示する
- 同意履歴をユーザー単位で保存し、同意済み文面と同意日時を後から確認できるようにする
- 文面の版が変わった場合、既存同意とは別に再同意を求める

## 管理モデル

### ConsentTerm

同意文面の正本。

- `title`: 表示名
- `body`: 利用者へ提示する本文
- `version_label`: 文面の版
- `active`: 現在有効か
- `consent_scope`: `global`, `project`, `document`, `download`, `shared_link`
- `requirement_timing`: global 同意向けの既存タイミング

同じ `title` / `version_label` の重複は許可しない。

### ProjectConsentSetting

Project 単位で、どの文面をどのタイミングで必須にするかを表す。

- `project`
- `consent_term`
- `required_on`: `first_access`, `download`, `shared_link_access`, `shared_link_download`
- `enabled`

初期実装で実際に enforcement するのは `first_access` と `download`。`shared_link_*` は共有リンク実装時に同じ設定モデルを使うための予約値とする。

### UserConsent

利用者が同意した履歴。

- `user`
- `consent_term`
- `consent_term_version_label`
- `target`: global では `nil`、Project 必須同意では Project
- `consented_at`
- `ip_address`, `user_agent`

`consent_term_version_label` を保存し、同じ文面タイトルでも版が変わった場合は別同意として扱う。

## Enforcement

初期実装では次の入口で同意を確認する。

- Project 詳細
- Project 配下の文書一覧
- 文書詳細
- DocumentVersion 詳細
- Project site / Docusaurus HTML 配信
- DocumentFile 直接ダウンロード
- DocumentVersion ZIP ダウンロード
- Project 文書一括 ZIP ダウンロード

権限判定を通過した後、同意が不足している場合だけ同意画面へリダイレクトする。権限がない利用者に同意画面だけを表示することはしない。

## 利用者画面

- 未同意の場合は `GET /consents/new` で不足している文面を表示する
- `POST /consents` で同意履歴を保存する
- `GET /consents` で現在有効な文面と自分の同意履歴を確認できる

## 管理画面

admin は次を管理できる。

- `admin/consent_terms`: 同意文面の作成・編集・無効化/削除
- `admin/project_consent_settings`: Project ごとの必須同意設定

## 現時点の制約

- 共有リンク自体はまだ実装されていないため、`shared_link_access` / `shared_link_download` は予約値に留める
- 同意後に自動で再実行できない POST ダウンロードは、安全な GET 画面へ戻して利用者に再操作してもらう
- 文面の法務レビューや契約本文の正当性はアプリ外の運用で担保する
