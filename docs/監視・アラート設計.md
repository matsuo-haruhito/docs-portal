# 監視・アラート設計

この文書は issue `#159` に対応する、`docs-portal` の監視・アラート設計の現行方針です。

## 1. 監視対象の分類

監視対象は次の 5 系統に分けます。

1. Rails アプリ
2. DB
3. storage
4. import / build / job
5. 外部依存

## 2. Rails アプリ監視

最低限見る指標:

- 5xx rate
- 4xx rate
- response time
- login failure

特に alert 対象にするもの:

- 5xx の急増
- ログイン不能
- 文書 HTML 表示または添付配信の広範囲失敗

## 3. DB 監視

最低限見る項目:

- 接続可否
- 接続数
- 遅いクエリ
- DB 容量

production で primary / queue / cache / cable を分ける場合は、primary 以外も同様に監視します。

## 4. storage 監視

対象:

- `document_files`
- `docs_sites`
- `imports`

見る項目:

- 使用量
- file not found 件数
- 権限エラー
- GCS 利用時の API 失敗

`DocumentFile#absolute_path` と importer の attachment/build copy は、ファイル欠落時に明示的なエラーを返すため、その失敗件数は監視対象に含めます。

## 5. import / build / job 監視

### import

現行 repo では `PublishJob` が import 状態を持っています。

見る項目:

- pending 件数
- failed 件数
- 最終成功 import 時刻
- `log_message` の失敗理由

### Docusaurus build

見る項目:

- build failed 件数
- 最終成功 build 時刻
- Kroki 未設定 / Kroki 接続失敗

### 将来の job 監視

job 基盤導入後は次を対象にします。

- failed jobs
- retry count
- queue latency
- stuck jobs

運用で管理画面を使うときは、次の切り分けを基準にします。

- 定期起点の失敗確認は `定期ジョブ`
- 生成依頼の入口確認は `生成ファイルイベント`
- 実際の生成処理と retry 履歴の確認は `生成ファイル実行履歴`

詳細な画面の見方と retry 導線は [生成ファイル再試行と定期ジョブ管理 runbook](./生成ファイル再試行と定期ジョブ管理runbook.md) を参照します。

自動リトライを検討する前の処理別リスクは [自動リトライ安全性棚卸し](./自動リトライ安全性棚卸し.md) を参照します。

## 6. 外部依存監視

対象:

- GitHub Actions / GitHub 連携
- Kroki
- 将来の mail delivery
- 将来の webhook delivery

alert 対象:

- Kroki endpoint 不達
- import API 呼び出し失敗の継続
- mail / webhook の継続失敗

## 7. アラート重要度

### Critical

- ログイン不能
- DB 接続不能
- 本文または添付の配信が広範囲で失敗
- import / build の連続失敗で業務継続に影響

### Warning

- 単発の import failed
- Kroki 未設定または一時的不達
- storage 使用量の閾値超過
- queue 滞留

### Info

- 定期 backup 成功
- build 成功
- import 成功

## 8. 通知先

現時点で具体的な通知先実装は repo にありませんが、方針としては次を分けます。

- Critical: 即時通知
- Warning: 運用チャネル集約
- Info: ダッシュボードや定期レポート

## 10. 現時点の監視ルール

- まずはアプリ/DB/import/build/storage の基本健全性を優先監視する
- 外部依存の retry は無制限にしない
- 失敗原因を追える識別子を log に残す設計を優先する
- 監視だけ増やして runbook がない状態を避ける
- 自動リトライは import / build / mail / webhook の安全性を処理ごとに棚卸ししてから、対象処理 1 つ単位の issue に分ける
