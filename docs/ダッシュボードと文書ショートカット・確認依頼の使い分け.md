# ダッシュボードと文書ショートカット・確認依頼の使い分け

この文書は、`dashboard` から辿る日常導線の役割差をまとめる。

新しい運用ルールはここでは定義しない。current 実装を前提に、どの画面を何のために使うか、一般利用者と internal user で見える導線がどこで分かれるかを整理する。

1. 画面全体の正本は [閲覧画面とUI](./specs/閲覧画面とUI.md)
2. 権限や利用者種別の前提は [基本モデルと権限](./specs/基本モデルと権限.md)

## 導線の全体像

### 1. ダッシュボード

`GET /dashboard` は、次に触る案件や文書へ戻るための入口。

- summary card では `閲覧可能案件` `閲覧可能文書` `保存ショートカット` `保留中の申請` をまとめて確認する
- 一覧では `案件` `お気に入り` `後で読む` `最近見た文書` `最近更新された文書` を並べて、次の移動先を決めやすくする
- 画面上部の CTA から `案件一覧へ` と `文書ショートカット一覧` へ移動できる
- internal user には追加で `確認依頼` が表示され、画面下部の `社内向け導線` から `確認依頼一覧` `外部送付履歴` `管理ダッシュボード` へ進める

使いどころ:

- どの案件や文書から作業を再開するか決めたいとき
- お気に入りや最近見た文書から、すぐ本文へ戻りたいとき
- internal user が pending task や社内向け運用画面へ入りたいとき

### 2. 文書ショートカット

`GET /document_bookmarks` は、利用者ごとの近道を最低限の一覧で見直す画面。

- `お気に入り` は継続的に参照する文書を残すための置き場
- `後で読む` は今すぐは開かないが、あとで確認したい文書を一時的に溜める置き場
- `最近見た文書` は操作で追加する一覧ではなく、閲覧履歴ベースで直近の文書を再表示する
- `お気に入り` と `後で読む` は、この画面から `解除` できる

使いどころ:

- 利用者自身が、よく読む文書と一時保留の文書を分けたいとき
- dashboard では足りない件数を、まとめて見直したいとき
- 案件一覧へ戻る前に、直近の個人用ショートカットを整理したいとき

補足:

- 権限外になった文書は、bookmark や recent history に残っていても一覧には出さない
- 文書詳細の `お気に入りに追加` `後で読むに追加` から、この一覧へ積み上がる

### 3. 確認依頼

`DocumentApprovalRequest` は、文書コメントより一段正式に、担当者を指定して確認を依頼したいときの導線。

- 文書詳細の `正式な確認依頼` セクションから `タイトル` `確認相手` `内容` を入れて依頼を作成できる
- 文書単位の一覧には `タイトル` `依頼者` `確認相手` `状態` が出る
- `GET /document_approval_requests` の全体一覧は internal user 向けで、`日時` `文書名` `依頼名` `依頼者` `確認相手` `状態` を見直せる
- 詳細画面では `状態` `依頼者` `確認相手` `対応者` `OK日時` `Cancel日時` を確認できる
- pending の依頼は、internal user が `OK`、internal user または依頼者本人が `Cancel` できる

使いどころ:

- 文書に対して、誰に確認してほしいかを明示して依頼を残したいとき
- internal user が、案件横断で pending の確認依頼を見直したいとき
- 依頼者が、自分の依頼が処理済みかを個別詳細で追いたいとき

注意点:

- dashboard 上で `確認依頼` への導線が見えるのは internal user だけ
- 一般利用者向けの個人導線は `文書ショートカット` が主で、`確認依頼一覧` を日常入口として前提にしない

## 迷ったときの切り分け

- まず文書を探したい: `案件一覧` または dashboard の `閲覧可能案件` から入る
- 自分用の近道を整理したい: `文書ショートカット` を使う
- 誰かにレビューや確認を正式に頼みたい: 文書詳細から `確認依頼` を作る
- internal user が案件横断で依頼や送付履歴を追いたい: dashboard の `社内向け導線` から各一覧へ進む
