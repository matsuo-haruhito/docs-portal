# 文書一覧の検索・実用フィルタ・ZIP出力 runbook

この文書は issue `#722` に対応する、`docs-portal` の文書一覧運用メモです。

## 1. この runbook が扱う画面

案件配下の文書一覧では、次の 3 つをまとめて扱います。

- `文書一覧`: `GET /projects/:project_code/documents`
- 左ペインの `文書ツリー`
- `選択した文書の最新版をZIPでダウンロード`: `POST /projects/:project_code/document_zip`

この画面は「閲覧可能な文書を探すこと」と「current project 内の文書をツリーから絞ること」と「見つけた文書を current page から、または current search/filter 条件に一致した文書全体からまとめて ZIP で持ち出すこと」を一つの流れで進める入口です。

別の役割を持つ近い画面は次です。

- 案件横断で文書を探したいとき: dashboard の `閲覧可能文書`
- 新しい ZIP を portal へ取り込みたいとき: [ZIPインポートdry-run運用 runbook](./ZIPインポートdry-run運用runbook.md)
- 手動アップロード後の `OK` / `NG` 判断をしたいとき: [手動アップロード差異確認runbook](./手動アップロード差異確認runbook.md)

## 2. 最初の切り分け順

1. 今いる案件の中でフォルダ名や元パスに心当たりがあるなら、左の `文書ツリー` で文書名・slug・元パスを頼りに絞る
2. 案件内の候補を一覧で広めに見たいときは `キーワード` で文書名、URL識別子、元パスに心当たりがないか探す
3. 候補が多いときは `タグ` `カテゴリ` `ファイル種` `公開範囲` で絞る
4. HTML の有無、添付の有無、PDF、図面系文書だけを見たい場合は `実用フィルタ` を足す
5. 一覧の `ヒット理由` `HTML` `添付` `最新版` を見て、表示中ページだけを選ぶか、`検索結果N件を全選択` で current search/filter 条件全体を選ぶかを決める
6. `ZIP出力オプション` を決めてダウンロードする
7. 0 件のときは条件を外すか、必要なら権限を確認する

## 3. 一覧の検索・絞り込みの意味

### キーワード

`キーワード` は current code で、少なくとも次に一致した文書を探します。

- 文書名
- URL識別子
- 元パス
- 版ラベル
- 本文
- 文書キーワード
- 添付ファイル名 / 添付ファイルのパス

最初からタグやカテゴリを決め打ちするより、迷ったときはキーワードから入る方が早いです。特に `元パス` や添付ファイル名まで届くので、文書名が曖昧でも探せることがあります。

### タグ / カテゴリ / ファイル種 / 公開範囲

- `タグ`: 運用上の分類語が分かっているときに使う
- `カテゴリ`: current `Document.categories` の分類で狭めたいときに使う
- `ファイル種`: Markdown、PDF、Office 文書など、文書の種別で絞りたいときに使う
- `公開範囲`: 閲覧前提の違いを見たいときに使う

キーワードで候補を出してから、これらを 1 つずつ足すと、何が効いているかを見失いにくくなります。

### 実用フィルタ

`実用フィルタ` は、検索語よりも「今ほしい状態」で絞るためのチェックです。

- `HTML生成済み`: そのまま HTML 本文を開いて確認したいとき
- `添付あり`: 添付ファイル込みで確認や配布をしたいとき
- `PDFあり`: PDF 付き文書だけを先に集めたいとき
- `図あり`: `puml` `plantuml` `d2` `mmd` `mermaid` を含む図面系文書を見たいとき

## 4. 左の文書ツリーの使い分け

### どんなときに使うか

左ペインの `文書ツリー` は、今開いている案件の中だけで文書を探したいときの近道です。案件をまたいで探す用途ではなく、current project の文脈を保ったまま、深い階層や似た文書名を絞りたいときに向きます。

### 絞り込み入力で見られるもの

ツリー上部の入力は current code で、少なくとも次を頼りに絞り込みます。

- 文書名
- slug
- 元パス

文書一覧の `キーワード` と違い、ツリー側は current project 内の階層ナビゲーションを助けるための入力です。案件横断検索や本文・添付ファイル名まで含む広い探索は、一覧側の `キーワード` を使います。

### `絞り込む` / `解除` の意味

- `絞り込む`: 現在の案件ツリーを、入力語に一致した文書だけへ絞る
- `解除`: ツリー絞り込みだけを外し、案件全体の階層表示へ戻る

絞り込み中でも `案件の文書ツリーを開く` / `閉じる` は今の条件を保ったまま動くので、hit が 1 件でも深いフォルダにいるときは開閉を併用すると見つけやすくなります。

### hit 件数と 0 件表示の読み方

- `「...」に一致する文書を N 件表示しています。`: current project の中で hit した件数
- `「...」に一致する文書はありません。文書名、slug、元パスで絞り込めます。`: ツリー側の手がかりでは見つからない状態

0 件のときは、元パスの一部や slug で短くし直すか、一覧側の `キーワード` に切り替えて本文・添付ファイル名まで含めて探す方が早いことがあります。

## 5. 一覧の読み方

### ヒット理由

`ヒット理由` には、検索語がどこに一致したかが badge 付きで出ます。ここを見ると、文書名に当たったのか、元パスなのか、添付ファイル名なのかを短く切り分けられます。

### 最新版 / HTML / 添付

- `最新版`: ZIP 出力の対象になる current latest version
- `HTML`: latest version に公開用 HTML があると `あり`
- `添付`: 版配下のファイル数。`なし` なら Markdown 原本だけの文書かもしれません

HTML の有無と添付数を一緒に見ると、「ブラウザ確認向きか」「配布物を ZIP で持っていくべきか」を判断しやすくなります。

### detail から一覧へ戻る

一覧の文書名 link は current URL を `return_to` として detail へ渡します。検索語、タグ、カテゴリ、ファイル種、公開範囲、実用フィルタ、ページを付けた一覧から detail へ入った場合、detail 末尾の `文書一覧へ戻る` はその一覧条件へ戻ります。

左ペインの `文書ツリー` や URL 直打ちなど、一覧 link 以外から detail へ入った場合は通常の案件配下 `文書一覧` が戻り先です。外部 URL や `//` で始まる unsafe な `return_to` は使わず、controller 側で既定の一覧へ戻します。

### 0 件時の見方

0 件のときは画面側でも次を案内します。

- キーワードを短くする
- 別名、ファイル名、タグで検索し直す
- タグ、カテゴリ、ファイル種、公開範囲、実用フィルタを外して再検索する

external user では、必要な文書が見えない場合に権限確認を依頼する案内も出ます。

### disabled checkbox の意味

一覧の先頭 checkbox は、ZIP に含める対象を選ぶためのものです。checkbox が disabled の行は、current latest version が無く、この一覧からは ZIP 出力対象にできません。

## 6. ZIP出力オプションと選択モードの使い分け

### ZIP内パス

- `文書名準拠`: ZIP 内のディレクトリ名を文書名ベースで揃えたいとき
- `元パス準拠`: portal へ入る前の source path に近い並びで持ち出したいとき

後で手元展開して見比べる前提なら `元パス準拠`、受け渡し相手に分かりやすい名前を優先したいなら `文書名準拠` が向きます。

### 含めるファイル

- `Markdown原本を含める`: 編集元の Markdown / MDX も一緒に出したいとき
- `添付ファイルを含める`: 添付 PDF や補足ファイルも同梱したいとき
- `PDFだけ出力`: PDF を中心に持ち出したいとき

全部を常に付けるより、「閲覧確認用なのか」「配布用なのか」でチェックを変える方が ZIP を小さく保てます。

### 表示中ページだけを選ぶとき

- `このページを全選択`: 表示中ページの checkbox だけをまとめて選ぶ
- `選択解除`: 現在の選択を戻す
- `0件選択中`: 表示中ページの中で現在選んでいる件数

1 ページ内だけで足りるときや、検索結果の一部だけを持ち出したいときは、このページ単位の選択が向いています。ページをまたぐ文書を一度に選ぶモードではありません。

### current search/filter 条件全体を選ぶとき

- `検索結果N件を全選択`: 今の `キーワード` `タグ` `カテゴリ` `ファイル種` `公開範囲` `実用フィルタ` に一致した、ZIP へ入れられる文書全体を選ぶ
- `N件選択中（検索結果全体）`: matching selection が有効で、表示中ページ以外も含めて選択対象になっている状態
- 個別の checkbox を触る: matching selection から離れて、現在の明示選択に戻る

複数ページにまたがる検索結果をまとめて持ち出したいときは、`検索結果N件を全選択` を使います。条件を変えると対象も変わるので、再検索したあとは件数表示を見直してから ZIP を作る方が安全です。

### current support の境界

ZIP 化されるのは、案件配下で current user が閲覧可能かつダウンロード可能な文書の latest version です。

- `検索結果N件を全選択` を使っても、権限外文書や `latest_version` なし文書は server-side で除外されます
- 一覧上でも `latest_version` がない文書の checkbox は disabled のままです
- external user で `view` 権限だけの文書は、選択しても ZIP には含まれません

## 7. 迷ったときの戻り先

- 案件横断で文書を探したい: [ダッシュボードと文書ショートカット・確認依頼の使い分け](./ダッシュボードと文書ショートカット・確認依頼の使い分け.md)
- 今いる案件の中で階層をたどりながら文書を探したい: この画面の左ペイン `文書ツリー` を使う
- 文書の公開モデルや latest version の前提を確認したい: [アプリケーション仕様](./アプリケーション仕様.md)
- ZIP を portal に取り込む運用を見たい: [ZIPインポートdry-run運用 runbook](./ZIPインポートdry-run運用runbook.md)
- 手動アップロード後の候補版を `OK` / `NG` で判断したい: [手動アップロード差異確認runbook](./手動アップロード差異確認runbook.md)

## 8. current support の境界

- この runbook は `project documents index`、左ペイン `文書ツリー`、`project document zip` の current UI だけを扱います
- ツリー絞り込みは current project 内の文書名 / slug / 元パスに閉じた導線で、案件横断検索や本文全文検索は再定義しません
- ZIP の選択モードは、表示中ページ単位の選択と、current search/filter 条件全体の matching selection までを扱います
- 検索ロジックの内部実装や ranking ルールそのものは再定義しません
- ZIP import、Git 連携 import、外部フォルダ同期の取り込み運用は別 runbook を正本にします
- upload panel の drag and drop 後に開く差異確認画面、`OK` / `NG`、反映後の rollback は [手動アップロード差異確認runbook](./手動アップロード差異確認runbook.md) を正本にします
