# 社外ユーザー向け情報露出点検チェックリスト

この文書は issue `#158` に対応する、社外ユーザー視点の情報露出点検用チェックリストです。

## 1. 点検の目的

- 社外ユーザーが権限外の文書を見られないことを確認する
- 画面、検索、関連情報、カタログ、ダウンロード導線で不要な情報が漏れないことを確認する
- Q&A / コメント、外部送付履歴、AI context export、文書ショートカットなど、追加された導線でも同じ境界を確認する
- 仕様上許容する metadata 露出と、許容しない露出を分けて確認する

## 2. 前提

- external / company_master_admin は、ProjectMembership と DocumentPermission に従って閲覧する
- `internal_only` 文書は社外ユーザーに露出しない
- 権限のない URL 直打ちは `403` または `404` のいずれかに統一する
- 仕様上、DevTools で見えるタイトルや slug 程度の metadata は当面許容範囲として扱う
- 外部向けに見せる public thread と、internal-only の確認事項や運用メモは分けて点検する
- AI context export や ZIP / download は、画面に出ている候補だけでなく、export / 配信される payload が current viewer の閲覧権限内に閉じることを確認する
- admin / integration 運用画面に出る raw path、payload、外部サービス識別子、webhook header は、社外ユーザーの権限外露出とは別観点として [運用 metadata 情報露出点検チェックリスト](./運用metadata情報露出点検チェックリスト.md) で確認する

## 3. チェック項目

### 画面

- 案件一覧で権限外の Project が出ない
- 文書一覧で権限外の Document が出ない
- 文書詳細で権限外の本文や添付導線が出ない
- Docusaurus site で権限外ページへのリンクが残らない

### 検索

- 本文検索で権限外文書がヒットしない
- タイトル、slug、版、本文、添付ファイル名の候補に権限外情報が出ない
- 検索結果の件数や並び順から、権限外文書の存在が推測しにくい

### 関連情報

- 関連文書に権限外文書が出ない
- カタログや文書セットに権限外文書が出ない
- bookmark / read confirmation / dashboard 表示が権限外情報を増やさない
- 文書ショートカットの `お気に入り` / `後で読む` / `最近見た文書` が current user から readable な文書だけを表示する

### 配信

- DocumentFile の直接配信で権限外ファイルが取得できない
- zip や archive 生成で権限外ファイルが混ざらない
- asset metadata に許容外の情報が入らない
- 外部送付履歴の一覧・検索・詳細が current user の可視範囲を広げない
- AI context export の HTML preview では、`含まれる文書` と `除外された文書`、選択済み文書、検索候補、50 件 preview 上限を分けて確認する
- AI context export の `document_q` や `選択済みだけ表示` は checkbox 候補の表示補助として読み、JSON / Markdown の export scope は `document_ids[]` と preview 後の出力対象件数で確認する
- AI context export の JSON / Markdown に、HTML preview の `除外された文書` が本文・添付 metadata として含まれない

### 追加導線

- Q&A / コメントは public Q&A と internal-only 確認事項を分けて点検する
- external user に internal-only の `note` / `issue` / `request_change` や位置メモが表示されない
- public Q&A への internal user の返信は public thread として扱われるため、外部に見せてよい内容かを確認する
- メール送付、共有リンク、外部共有ビューの preview がある場合は同じ基準で点検する
- AI context export は外部 AI サービスへの自動送信ではなく、current viewer が download する JSON / Markdown の中身として点検する

## 5. 代表ケース

次の request spec を代表ケースとして確認する。

- `spec/requests/external_document_access_spec.rb`
- `spec/requests/document_search_spec.rb`
- `spec/requests/project_site_security_spec.rb`
- `spec/requests/document_files_spec.rb`
- `spec/requests/document_sites_spec.rb`
- `spec/requests/document_review_comments_spec.rb`
- `spec/requests/project_ai_contexts_spec.rb`
- `spec/requests/document_delivery_logs_spec.rb`
- `spec/requests/document_bookmarks_spec.rb`

日常 smoke として同じ集合をまとめて実行する場合は、次の軽量入口を使う。

```sh
bundle exec ruby bin/external_user_exposure_smoke
```

PR / release / 運用引き継ぎで短い evidence を残す場合は、同じ smoke を `--format markdown` 付きで実行する。

```sh
bundle exec ruby bin/external_user_exposure_smoke --format markdown
```

Markdown digest は、smoke 名、実行時刻、RSpec 結果、代表 spec、対象 surface、確認目的、次に見る runbook だけを残す。HTML / JSON / ZIP payload 本文、raw response、token-like value、文書タイトル、添付 metadata、権限外文書名は貼らない。admin / integration 運用画面の raw path、payload、外部サービス識別子は、この digest ではなく [運用 metadata 情報露出点検チェックリスト](./運用metadata情報露出点検チェックリスト.md) と `bin/operational_metadata_exposure_smoke` 側で扱う。

`spec/docs/exposure_smoke_checklist_drift_spec.rb` は、この代表ケース一覧と `bin/external_user_exposure_smoke` の `SPEC_FILES` が一致していることを source-level guard として確認する。代表 spec を追加・削除する場合は、smoke script とこの節を同じ PR で更新し、admin / integration metadata の代表 spec はこの一覧へ混ぜず [運用 metadata 情報露出点検チェックリスト](./運用metadata情報露出点検チェックリスト.md) 側で扱う。

この smoke は社外ユーザーの可視範囲確認に閉じる。admin / integration 運用画面の raw path、payload、Graph metadata 点検は [運用 metadata 情報露出点検チェックリスト](./運用metadata情報露出点検チェックリスト.md) 側で扱う。

## 6. 定期点検の手順

1. external ユーザーでログインする
2. 案件一覧、文書一覧、文書詳細、検索を順に確認する
3. 権限外 URL を直接開く
4. 添付ファイル、zip、site 配信を確認する
5. Q&A / コメント、文書ショートカット、外部送付履歴、AI context export など追加導線を確認する
6. AI context export では HTML preview の `含まれる文書` / `除外された文書`、選択済み文書、検索候補、JSON / Markdown payload を突き合わせる
7. JSON / Markdown / ZIP などの出力がある場合は、画面表示だけでなく出力 payload に権限外文書が含まれないことを確認する
8. 関連する request spec を再実行する

## 7. 合格基準

- 権限外文書の本文が見えない
- 権限外文書の直接導線が出ない
- 権限外情報が検索、関連、カタログ、配信で増えない
- public Q&A と internal-only 確認事項が混線しない
- AI context export、ZIP、外部送付履歴、文書ショートカットが current user の可視範囲を広げない
- 例外的に許容する metadata が明文化されている
