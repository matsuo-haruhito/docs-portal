# アプリケーション仕様

## 概要

- 本アプリは、案件ごとの文書公開、版管理、閲覧権限制御、添付ファイル配布を行う Rails ポータルである。
- Markdown 文書は Docusaurus build を公開物として扱い、Rails 側では閲覧権限を付けたうえで配信する。

## ユーザー種別

- `internal`
  - 社内ユーザー。
  - 全案件を閲覧可能。
  - 管理画面へアクセス可能。
  - 会社マスタも管理可能。
- `external`
  - 社外ユーザー。
  - 自身に紐づく案件または権限付与された文書のみ閲覧可能。
  - 管理者相当の概念を将来追加しても、会社マスタにはアクセスさせない。

## 認証

- セッション認証を使う。
- 未ログイン時はログイン画面へリダイレクトする。
- ナビゲーション右上からログアウトできる。

## 画面

- ブランド名クリックで TOP に戻る。
- 管理画面は `internal` ユーザーのみ表示する。
- 文書詳細では `tree_view` gem を使って「案件 > 文書」の左ペインツリーを表示する。
- `external` ユーザーが参照できる Project は `project_memberships` がある案件だけとし、案件詳細・文書一覧でも同じ制約を適用する。
- 管理画面では以下を CRUD 可能とする。
  - 会社
  - ユーザー
  - 案件
  - 案件所属
  - 文書
  - 文書権限

## 文書表示

- Markdown 公開物は Docusaurus build を版単位で保存する。
- Rails 側では Docusaurus HTML の内部リンクと静的 asset の参照先を版配下ルートへ張り替えて配信する。
- そのため、左ペイン、パンくず、標準ドキュメントレイアウトは Docusaurus 標準の見た目を利用する。
- `DOCUSAURUS_EDIT_URL` を build 時に与えた場合は、`Edit this page` を表示できる。
- 文書詳細の「表示」リンクは、対応する Docusaurus 生成 HTML が実在する版にだけ表示する。
- Markdown 添付は Docusaurus 画面とは別系統の生ファイル配信とし、text 系は UTF-8 charset 付きで返す。
- 添付ファイル配信は `download` 権限を要求し、`view` 権限だけでは取得できない。

## import

- internal import API が読み取るローカルパスは `storage/imports/` 配下に限定する。
- import manifest の添付ファイル `storage_key` は相対パスのみ許可し、絶対パスや path traversal は拒否する。

## seed データ

- seed 用 CSV は `db/data/` に置く。
- `db/seeds.rb` は `upsert_all` ベースで投入し、初期化前提の `delete_all` は行わない。
- sample 文書ファイルは `storage/document_files/external_samples/` 配下を正本として扱う。
- 配置規約は `storage/document_files/external_samples/<sample-set>/<document-dir>/...` とし、`document-dir` ごとに 1 文書として取り込む。
- `sample-set` と `document-dir` の命名は利用者に委ねる。seed 時には `sample-set` ごとに Project、`document-dir` ごとに Document / DocumentVersion / DocumentFile を生成する。
- `document-dir` 配下に Markdown がある場合は seed 時に Docusaurus build を行い、対応する `DocumentVersion.site_build_path` と `storage/docs_sites/<version.id>/...` を生成する。
- seed 用 Markdown は build 成功を必須とし、失敗した場合は seed 全体を失敗させる。
