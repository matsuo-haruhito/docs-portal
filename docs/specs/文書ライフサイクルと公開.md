# 文書ライフサイクルと公開

この文書は、Document / DocumentVersion / DocumentFile の状態、公開、表示ルールの正本です。

## Data Classification

- `visibility_policy` と `data_classification_tags` は分けて扱う
- 閲覧可否・download 可否・社外露出可否は `visibility_policy` / ProjectMembership / DocumentPermission で判定する
- `data_classification_tags` は権限判定そのものには使わず、取り扱い補助のためのタグとして使う
- 初期対象は Document 単位のみとし、DocumentVersion / DocumentFile / DocumentSet / Catalog 単位のタグは `#1066` で責務境界と first slice を整理してから扱う
- import dry-run では `DocumentClassificationSuggester` により source path / file extension / frontmatter から category, document_kind, visibility_policy, snapshot_kind を推定できる

## DocumentVersion 品質チェック

- `DocumentVersionQualityChecker` により、版単位の品質チェックを都度計算できる
- 初期導線は `GET /document_versions/:public_id/quality_check`
- Markdown 原本が添付されている版では、相対 Markdown link / image / attachment 参照切れも error として検出する
- Docusaurus build 成果物の entry が見つからない場合は、build failure 相当の error として扱う

## 文書重要度

- `Document` は `importance_level` と `recommended_sort_order` を持つ
- `importance_level` は `critical`, `important`, `normal`, `reference`, `deprecated` を持つ
- Project 詳細と文書一覧では、文書を `importance_level`, `recommended_sort_order`, `title`, `id` の順で優先表示する

## DocumentVersion の状態

### draft

- 社内ユーザーおよび admin は閲覧できる
- external / company_master_admin には表示しない
- `latest_version` 候補にしない

### published

- 公開 window 内であれば通常導線に出せる
- `published_at` を持つ
- `latest_version` 候補にできる

### archived

- 通常導線では表示しない
- 社内ユーザーおよび admin は必要に応じて参照できる
- `latest_version` 候補にしない

## latest_version

- `latest_version` は、原則として `published` な `DocumentVersion` のうち、もっとも新しく作成されたものを指す
- `version_label` は表示・識別用であり、最新版判定の正本にはしない
- semantic version として機械比較できることを前提にしない
- `draft` / `archived` な `DocumentVersion` は `latest_version` の候補にしない

## 文書表示

- 文書詳細では、最新版、版一覧、添付、タグ、関連文書、Q&A、確認依頼、社内レビューを表示できる
- DocumentVersion 詳細では、公開状態、公開期間、ソースコミット、添付・元ファイル、品質チェック導線を表示する
- text 系ファイルは Rails 側の preview を使い、Markdown / MDX は Docusaurus viewer、PDF / Office / image は配布ファイルとして扱う

## ドキュメントサイト権限フィルタ

- project site / document site 上の内部 link は、現在 viewer が参照できない DocumentVersion を指す場合に除外する
- 権限外ページへの直接アクセスは 403 にする
- Docusaurus asset / client bundle 経由で slug や一部 metadata が技術的に見える可能性は許容する

## seed データ

- seed は `db/data/*.csv` に加えて、`storage/document_files/external_samples/` 配下のサンプル文書も扱う
- Markdown 群を取り込む場合、seed 中に Docusaurus build を実行し、`storage/docs_sites/<version.id>/...` を生成する
- build 失敗時は `db:seed` を失敗させる

## AccessLog

- HTML view、DocumentFile download、ZIP download、preview 実行などの主要操作を監査ログに残す
- JS / CSS / image asset は通常の閲覧履歴としては記録対象外にする
- `last_login_at` は `users` 側で別管理する
