# 文書ライフサイクルと公開

この文書は、Document / DocumentVersion / DocumentFile の状態、公開、表示ルールの正本です。

## Data Classification

- `visibility_policy` と `data_classification_tags` は分けて扱う
- 閲覧可否・download 可否・社外露出可否は `visibility_policy` / ProjectMembership / DocumentPermission で判定する
- `data_classification_tags` は権限判定そのものには使わず、取り扱い補助のためのタグとして使う
- `data_classification_tags` は DLP、法務判定、承認 workflow、既存文書や添付の一括分類移行の正本にはしない
- 初期対象は Document 単位のみとし、DocumentVersion / DocumentFile / DocumentSet / Catalog 単位の固有タグは後続候補として扱う
- DocumentVersion / DocumentFile は、first slice では親 Document の `data_classification_tags` を継承して表示・警告に使う候補を優先する
- version / file 固有タグを許す場合も、`visibility_policy` / ProjectMembership / DocumentPermission へ混ぜず、取り扱い補助としてだけ扱う
- import dry-run では `DocumentClassificationSuggester` により source path / file extension / frontmatter から category, document_kind, visibility_policy, snapshot_kind を推定できる

### DocumentVersion / DocumentFile への拡張候補

- first implementation は DocumentVersion / DocumentFile / DocumentSet / Catalog のうち 1 model または 1 screen/surface に絞る
- 候補 A: DocumentFile の一覧・詳細で、親 Document の分類タグを表示補助として見せる
- 候補 B: DocumentVersion quality check で、添付・元ファイルの分類が親 Document と矛盾しそうな場合に warning を出す
- 候補 C: import dry-run / 手動 upload のどちらか 1 導線で、分類タグの推定結果または継承元を確認できるようにする
- ZIP download / 外部送付履歴 / DocumentSet / Catalog への展開は、DocumentFile または DocumentVersion の first slice 後に必要性を再確認する
- 後続 issue は、表示補助、quality warning、import/upload 確認、ZIP/download/送付履歴のいずれか 1 つに分ける

### 禁止境界

- classification tag を閲覧可否、公開可否、download 可否の判定条件にしない
- classification tag によって `external` / `company_master_admin` / internal user の可視範囲を変更しない
- classification tag を `DocumentPermission` や `ProjectMembership` の代替にしない
- version / file 固有タグを導入する場合も、既存 Document の分類を一括移行しない

## DocumentVersion 品質チェック

- `DocumentVersionQualityChecker` により、版単位の品質チェックを都度計算できる
- 初期導線は `GET /document_versions/:public_id/quality_check`
- Markdown 原本が添付されている版では、相対 Markdown link / image / attachment 参照切れも error として検出する
- Docusaurus build 成果物の entry が見つからない場合は、build failure 相当の error として扱う
- 品質チェック結果は read-only evidence であり、公開承認 gate や正式レビュー承認 workflow の state ではない。workflow として扱う必要が出た場合は [正式レビュー承認 workflow 境界メモ](../正式レビュー承認workflow境界メモ.md) に戻す

## 文書重要度

- `Document` は `importance_level` と `recommended_sort_order` を持つ
- `importance_level` は `critical`, `important`, `normal`, `reference`, `deprecated` を持つ
- Project 詳細と文書一覧では、文書を `importance_level`, `recommended_sort_order`, `title`, `id` の順で優先表示する

## 多言語文書

- 多言語文書の first slice は、`Document` 単位の `language` と `Document` 間の manual translation relation を候補正本にする
- `DocumentVersion` 単位の `language`、版ごとの翻訳対応、翻訳版ごとの差分管理は後続 issue で扱う
- 翻訳元 / 翻訳先の関連は `Document` 間で表現し、各翻訳 `Document` は自分の `latest_version` / `published` / `archived` / 権限判定を独立して持つ
- `latest_version`、公開 window、archive / restore、DocumentPermission / ProjectMembership は翻訳間で共有しない
- viewer / 文書一覧では、first slice では閲覧可能な翻訳候補の存在を表示する範囲に閉じる
- 翻訳候補を表示する場合も、現在の利用者が閲覧できない `Document` への link や metadata は露出しない
- 検索 index 多言語最適化、コメント翻訳、機械翻訳、自動翻訳生成、Docusaurus i18n 全面移行、既存文書一括移行はこの first slice に含めない
- 後続実装 issue を切る場合は、DB migration + admin manual relation + 最小表示確認を 1 lane として扱い、model change と viewer / list UI change が広がる場合は分割する

## DocumentVersion の状態

DocumentVersion の状態は、表示可否、latest version 候補、公開 window の読み方を決めるための公開状態です。法務・顧客承認、公開前承認 route、承認者 chain、通知、差し戻しを含む正式レビュー承認 workflow ではありません。

### draft

- 社内ユーザーおよび admin は閲覧できる
- external / company_master_admin には表示しない
- `latest_version` 候補にしない

### published

- 公開 window 内であれば通常導線に出せる
- `published_at` を持つ
- `latest_version` 候補にできる
- `published` は公開状態を表す値であり、顧客承認済み、法務承認済み、正式 workflow 完了済みを自動的には意味しない

### archived

- 通常導線では表示しない
- 社内ユーザーおよび admin は必要に応じて参照できる
- `latest_version` 候補にしない

## latest_version

- `latest_version` は、原則として `published` な `DocumentVersion` のうち、もっとも新しく作成されたものを指す
- `version_label` は表示・識別用であり、最新版判定の正本にはしない
- semantic version として機械比較できることを前提にしない
- `draft` / `archived` な `DocumentVersion` は `latest_version` の候補にしない
- `latest_version` への昇格は表示対象の選択であり、承認 workflow の完了や送付承認を意味しない

## 文書表示

- 文書詳細では、最新版、版一覧、添付、タグ、関連文書、Q&A、確認依頼、社内レビューを表示できる
- DocumentVersion 詳細では、公開状態、公開期間、ソースコミット、添付・元ファイル、品質チェック導線を表示する
- text 系ファイルは Rails 側の preview を使い、Markdown / MDX は Docusaurus viewer、PDF / Office / image は配布ファイルとして扱う

## ドキュメントサイト権限フィルタ

- project site / document site 上の内部 link は、現在 viewer が参照できない DocumentVersion を指す場合に除外する
- 権限外ページへの直接アクセスは 403 にする
- Docusaurus asset / client bundle 経由で slug や一部 metadata が技術的に見える可能性は許容する

## seed データ

- seed は `db/data/*.csv` に加えて、`storage/document_files/external_samples/` 配下のサンプル文書も扱う
- Markdown 群を取り込む場合、seed 中に Docusaurus build を実行し、`storage/docs_sites/<version.id>/...` を生成する
- build 失敗時は `db:seed` を失敗させる

## AccessLog

- HTML view、DocumentFile download、ZIP download、preview 実行などの主要操作を監査ログに残す
- JS / CSS / image asset は通常の閲覧履歴としては記録対象外にする
- `last_login_at` は `users` 側で別管理する
