# 基本モデルと権限

この文書は、`docs-portal` の基本モデル、識別子、権限制御、申請系フローの正本です。

## 識別子

- DB 内部の主キー `id` は、外部公開 URL には原則使用しない
- アプリケーション管理テーブルには `public_id` を持たせる
- `public_id` は作成時に自動生成し、原則変更しない
- `public_id` は URL-safe な文字列とし、テーブルごとに prefix を付ける
- 通常の `create!` / `save!` 経路では model concern で自動採番する
- seed の `upsert_all` では callback が走らないため、`db/seeds.rb` 側で明示投入する
- 外部 URL では次を使う
  - Project: `code`
  - Document: Project 内の `slug`
  - DocumentVersion: `public_id`
  - DocumentFile: `public_id`
- 管理画面でも、外部公開 URL と同じく `public_id` / `code` / `slug` を優先して使う

## ユーザー種別

### internal

- 社内ユーザー
- current code の `user_type` としては `internal` / `external` / `company_master_admin` の 3 種のみを持つ
- `User#admin?` は `internal?` の alias であり、view / controller / helper では admin と呼ぶことがある
- 管理画面へアクセス可能
- 全案件・全文書を閲覧可能
- draft / published / archived の各 DocumentVersion を閲覧可能

### company_master_admin

- 会社マスタ管理者
- `/admin` へ直接入った場合は current `main` では `会社` 管理へ redirect される
- 自社のユーザーと会社マスタのメンテナンスを行える
- 他社の会社マスタ、他社ユーザー、案件、文書、文書権限は管理できない
- 文書閲覧権限は `external` と同じ制約に従う

### external

- 社外ユーザー
- 自身が所属する会社・案件・文書権限に基づき、許可された文書のみ閲覧可能
- 管理者相当の概念を追加しても、他社の会社マスタにはアクセスさせない

## 認証

- セッション認証を使う
- 未ログイン時はログイン画面へリダイレクトする
- ナビゲーション右上からログアウトできる
- ログイン成功時に `users.last_login_at` を更新する
- ログイン済みユーザーは自身の access request 一覧を参照できる

## Document visibility_policy

### internal_only

- `internal` のみ閲覧可能
- `external` / `company_master_admin` は DocumentPermission があっても閲覧不可

### restricted_external

- `internal` は閲覧可能
- `external` / `company_master_admin` は、ProjectMembership と DocumentPermission がある場合のみ閲覧可能

### public_with_login

- `internal` は閲覧可能
- `external` / `company_master_admin` は、ProjectMembership がある場合に閲覧可能
- DocumentPermission は不要
- 添付ファイル download は別判定とし、`DocumentPermission.download` がない限り取得できない

## DocumentPermission

### view

- Docusaurus HTML の閲覧が可能
- 添付ファイルは取得不可

### download

- Docusaurus HTML の閲覧が可能
- 添付ファイルを取得可能
- `download` は `view` を包含する

## AccessRequest

- `AccessRequest` は `Project` `Document` `DocumentFile` を対象にできる
- 利用者は自分の申請一覧を参照できる
- 利用者は pending な自分の申請を取り消せる
- 初回の利用者向け導線は、download 不可の文書/添付に対する download 申請を対象とする
- `internal` は管理画面から申請一覧を参照し、pending 申請を承認または却下できる

## DocumentApprovalRequest

- `DocumentApprovalRequest` は `Document` 単位の最小確認依頼である
- requester は対象 `Document` を閲覧できる active user とする
- approver は任意指定とし、初期実装では `internal` を想定する
- status は `pending`, `approved`, `cancelled` を持つ
- `approved` / `cancelled` 時は `acted_by` と時刻を記録する
- 文書詳細では確認依頼の作成導線を表示し、`internal` には文書単位一覧への導線も表示する
- `internal` は確認依頼一覧・詳細を参照でき、OK / Cancel を実行できる
- requester は自分が作成した確認依頼の詳細を参照できる

## 権限変更 dry-run

- 管理画面では、Project 単位の権限変更を適用する前に preview を取得できる
- preview は `internal` 限定の `GET /admin/projects/:code/permission_preview` とする
- 対象 viewer は `user_ids[]` と `company_ids[]` で指定できる
- preview は viewer ごとの閲覧可能文書の増減、download 可能文書の増減、company ごとの影響人数集計を返す
- ProjectMembership 付与/剥奪と DocumentPermission 相当の付与/剥奪を、変更前に dry-run で確認できる
- `internal_only` 文書は ProjectMembership を付与しても社外ユーザーの gained document に含めない

## 外部表示プレビュー

- 管理画面では、Project 単位で社外ユーザー視点の表示 preview を確認できる
- 初期導線は `admin/projects/:code/external_preview` とし、案件編集画面から遷移できる
- user 指定 preview は、指定した active な `external` / `company_master_admin` を viewer として既存 `viewable_by?` / `downloadable_by?` 判定を流用する
- company 指定 preview は、その会社に所属する active な `external` / `company_master_admin` を一覧し、viewer ごとの差分を同じ画面で確認する
- preview では文書の表示/非表示、添付ファイルの download 可否を確認できる
- preview 実行時は `AccessLog` に `target_type = external_preview` として監査ログを残す
- `external` 自身は preview 機能を利用できない
