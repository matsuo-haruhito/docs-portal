# PROC-DOC-DRAFT ChatGPTで資料ドラフトを作成する

| 項目 | 内容 |
|---|---|
| 手順ID | PROC-DOC-DRAFT |
| ユースケース | 提案資料作成、業務整理資料作成、要件整理、ドキュメント整備 |
| パターン | 資料ドラフト作成 |
| AIツール | ChatGPT |
| 主な目的 | メモ、議事録、既存資料、調査結果をもとに、資料や手順書のたたき台を作る |
| 主な成果物 | 章立て、ドラフト本文、比較表、確認事項、次アクション |

## 向いている状況

- 顧客情報、議事録、ヒアリングメモを整理したい。
- 提案書、業務整理資料、要件定義書、設計書、手順書のたたき台を作りたい。
- まだコード修正やPR作成までは不要。
- 文章の構成、粒度、表現を整えたい。
- 複数の観点を比較表やチェックリストにしたい。

## 向いていない状況

- 既存コードを横断的に調査する必要がある。
- リポジトリ内のファイルを直接修正したい。
- テストやCIを確認したい。
- GitHub上のPRやIssueと連動して作業したい。

その場合は、`PROC-CODE-GH-READ`、`PROC-CODE-LOCAL-EDIT`、`PROC-CODE-GH-PR` を検討します。

## 入力情報

| 入力 | 例 |
|---|---|
| 元情報 | 議事録、メモ、提案要件、調査メモ |
| 作りたい成果物 | 提案書、業務フロー、要件一覧、手順書 |
| 読者 | 顧客、社内PM、開発者、レビュー担当 |
| 粒度 | 1枚資料、詳細手順、一覧表、章立て |
| 制約 | ベストプラクティスではなくユースケース集にする、など |

## 手順

1. 作りたい成果物と読者を決める。
2. 元情報を貼り付ける、または要点を箇条書きで渡す。
3. まず章立てや一覧構成を作らせる。
4. 粒度、観点、表形式、文体を指定してドラフト化する。
5. 不足情報、確認事項、前提条件を抽出させる。
6. 必要に応じて、コード調査やPR化が必要な項目を別手順に切り出す。

## 依頼文例

```text
以下のメモをもとに、システム開発におけるAI活用ユースケース集のドラフトを作ってください。
ベストプラクティスではなく、実際に使える場面の棚卸しとして整理してください。
工程、ユースケース、利用AI、入力、手順、成果物、注意点を表形式にしてください。
```

## 判断ポイント

| 判断 | 切り替え先 |
|---|---|
| コードの実態確認が必要 | PROC-CODE-GH-READ または PROC-CODE-LOCAL-READ |
| そのままリポジトリを修正したい | PROC-CODE-LOCAL-EDIT または PROC-CODE-GH-PR |
| PR本文やREADMEに反映したい | Codex CLIまたはCodex Web/App |
| 判断フローや図も作りたい | PlantUML / Mermaid / D2 化を検討 |

## 成果物テンプレート

```markdown
# 資料ドラフト

## 目的

## 対象読者

## 前提

## 全体構成

## 本文

## 一覧表

## 判断ポイント

## 未確認事項

## 次アクション
```

## 注意点

ChatGPTで作成したドラフトは、業務ルール、既存システム、社内標準、顧客固有の前提と必ず照合します。コードやリポジトリとの整合が必要な場合は、コード解析手順を併用します。
