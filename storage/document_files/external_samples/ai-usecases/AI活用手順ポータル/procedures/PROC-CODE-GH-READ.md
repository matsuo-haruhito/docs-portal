# PROC-CODE-GH-READ GitHub Connectorで既存コードを調査する

| 項目 | 内容 |
|---|---|
| 手順ID | PROC-CODE-GH-READ |
| ユースケース | コード解析 |
| パターン | GitHub上のコード解析 |
| AIツール | ChatGPT + GitHub App Connector |
| 主な目的 | GitHub上の既存コードから仕様、呼び出し元、影響範囲、確認事項を把握する |
| 主な成果物 | 調査メモ、影響範囲一覧、確認事項、次アクション |

## 向いている状況

- ローカルにコードを持っていない。
- GitHub上のリポジトリを会話形式で調査したい。
- 修正ではなく、まず仕様や影響範囲を把握したい。
- Issue、PR、README、設定ファイルも合わせて確認したい。
- 調査結果をそのまま要件整理や設計資料に転用したい。

## 向いていない状況

- ローカルの未コミット差分を調べたい。
- テストを実行したい。
- 複数ファイルを実際に編集してコミットしたい。
- PR作成まで自動で進めたい。

その場合は、`PROC-CODE-LOCAL-READ`、`PROC-CODE-LOCAL-EDIT`、`PROC-CODE-GH-PR` を検討します。

## 入力情報

| 入力 | 例 |
|---|---|
| リポジトリ | matsuo-haruhito/work-codex |
| ブランチ | main |
| 調査対象 | 認証処理、ファイルアップロード、CSV import など |
| 関連Issue / PR | #10、#15 など |
| 知りたい観点 | 仕様、依存関係、影響範囲、テスト有無 |

## 手順

1. ChatGPTでGitHub Connectorが使える状態にする。
2. 対象リポジトリ、ブランチ、調査対象を指定する。
3. まず関連ファイル、README、設定ファイル、テストの場所を洗い出すよう依頼する。
4. 主要な処理の入口、呼び出し元、依存先、関連テストを確認する。
5. 現在の仕様、想定される影響範囲、注意点を整理する。
6. 修正が必要な場合は、Codex Web/AppまたはCodex CLIの手順に切り替える。

## 依頼文例

```text
GitHub Connectorで `owner/repo` の `main` ブランチを見て、
`〇〇機能` の現在の仕様を調査してください。
関連ファイル、呼び出し元、関連テスト、影響範囲、未確認事項を整理してください。
まだ修正は不要です。
```

## 判断ポイント

| 判断 | 切り替え先 |
|---|---|
| 調査だけで完了する | この手順で完了 |
| GitHub上で修正してPR化したい | PROC-CODE-GH-PR |
| ローカルでテストしながら修正したい | PROC-CODE-LOCAL-EDIT |
| ローカル未コミット差分も見たい | PROC-CODE-LOCAL-READ |

## 注意点

GitHub Connectorで参照できるのは、基本的にGitHub上の情報です。ローカルで未コミットの変更、生成済みファイル、手元だけにある設定は対象外になるため、必要に応じてCodex CLIに切り替えます。
