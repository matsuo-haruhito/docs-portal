# PROC-CODE-GH-PR Codex Web/AppでGitHub上の修正をPR化する

| 項目 | 内容 |
|---|---|
| 手順ID | PROC-CODE-GH-PR |
| ユースケース | 実装、Issue対応、PR作成 |
| パターン | GitHub上で修正してPR化 |
| AIツール | Codex Web/App |
| 主な目的 | GitHub上のIssueや要件をもとに、修正、コミット、PR作成、CI確認まで進める |
| 主な成果物 | ブランチ、コミット、PR、CI確認結果、レビュー依頼メモ |

## 向いている状況

- GitHub上で対象リポジトリ、Issue、PRを扱いたい。
- 修正後にPull Requestまで作成したい。
- ローカル環境を用意せず、Web/App上で作業を進めたい。
- 変更対象がGitHub上の最新コードに対するもの。
- CI結果やPRコメントも含めて確認したい。

## 向いていない状況

- ローカル未コミット差分を扱いたい。
- ローカル固有の設定やデータを使ってテストしたい。
- 大量生成ファイルやバイナリを細かく扱いたい。

その場合は、`PROC-CODE-LOCAL-EDIT` を検討します。

## 入力情報

| 入力 | 例 |
|---|---|
| リポジトリ | `owner/repo` |
| ベースブランチ | `main` |
| Issue / 要件 | `#123`、要件メモ、バグ報告 |
| 変更方針 | 既存仕様を保つ、テストを追加する、UI変更なし等 |
| 確認したいこと | CI結果、影響範囲、レビュー観点 |

## 手順

1. 対象リポジトリ、ベースブランチ、関連Issueを指定する。
2. Codex Web/AppにIssue内容と関連ファイルを調査させる。
3. 実装方針、変更予定ファイル、テスト方針を出させる。
4. 作業ブランチを作成させる。
5. 必要なファイルを修正させる。
6. 可能な範囲でテストや静的チェックを実行させる。
7. 変更概要、テスト結果、残課題をまとめさせる。
8. PRを作成させる。
9. CI/checkがある場合は結果を確認させる。
10. 失敗した場合はログを確認し、必要に応じて追加修正する。

## 依頼文例

```text
`owner/repo` の `main` ブランチをベースに、Issue #123 の対応をしてください。
まず関連ファイルと実装方針を調査し、必要な修正とテストを追加してください。
完了後、PRを作成し、CI結果と残課題をまとめてください。
```

## 判断ポイント

| 判断 | 対応 |
|---|---|
| 修正範囲が複数テーマに分かれる | PRを分割する |
| CIが失敗した | CIログを確認し、原因別に修正する |
| ローカルでしか再現できない | Codex CLIに切り替える |
| 仕様が曖昧 | PR作成前に確認事項を整理する |

## PR本文テンプレート

```markdown
## 概要

## 変更内容

## 確認したこと

## テスト

## 影響範囲

## 残課題

## 関連Issue
```

## 注意点

Codex Web/AppはGitHub上での修正とPR化に向いています。一方で、ローカル固有の環境、未コミット差分、ローカルデータを使った確認が必要な場合は、Codex CLIで作業したほうが安全です。
