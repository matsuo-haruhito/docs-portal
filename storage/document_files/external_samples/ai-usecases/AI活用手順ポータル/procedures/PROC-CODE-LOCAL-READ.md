# PROC-CODE-LOCAL-READ Codex CLIでローカルコードを調査する

| 項目 | 内容 |
|---|---|
| 手順ID | PROC-CODE-LOCAL-READ |
| ユースケース | コード解析 |
| パターン | ローカルコード解析 |
| AIツール | Codex CLI |
| 主な目的 | ローカルの作業コピー、作業ブランチ、未コミット差分を含めてコードを調査する |
| 主な成果物 | 調査メモ、関連ファイル一覧、差分影響メモ |

## 向いている状況

- 自分のPCまたは開発環境にコードがある。
- GitHubにpushしていない変更も含めて調べたい。
- ローカルの設定ファイル、生成物、実行ログも見たい。
- 実装前に関連箇所や影響範囲を把握したい。
- 調査後にそのまま修正やテストに進む可能性がある。

## 向いていない状況

- GitHub上のリポジトリを会話形式で軽く調べたいだけ。
- ローカル環境がない。
- PRやIssueの文脈を中心に調査したい。

その場合は、`PROC-CODE-GH-READ` または `PROC-CODE-GH-PR` を検討します。

## 入力情報

| 入力 | 例 |
|---|---|
| ローカルパス | `/work/project-name` |
| ブランチ | `feature/import-flow` |
| 調査対象 | 認証処理、一覧画面、CSV import など |
| 関連差分 | `git diff`、未コミットファイル |
| 知りたい観点 | 仕様、影響範囲、関連テスト、修正方針 |

## 手順

1. 対象リポジトリのローカルディレクトリでCodex CLIを起動する。
2. 現在のブランチ、未コミット差分、関連ファイルを確認させる。
3. 調査対象の機能名、画面名、API名、エラー内容などを伝える。
4. 関連ファイル、呼び出し元、関連テストを洗い出させる。
5. 現在の仕様、変更時の影響範囲、注意点をまとめさせる。
6. 修正に進む場合は `PROC-CODE-LOCAL-EDIT` に切り替える。

## 依頼文例

```text
このローカルリポジトリで、`〇〇機能` の実装箇所を調査してください。
未コミット差分も含めて、関連ファイル、呼び出し元、関連テスト、影響範囲を整理してください。
まだ修正はしないでください。
```

## 判断ポイント

| 判断 | 切り替え先 |
|---|---|
| 調査だけで完了する | この手順で完了 |
| ローカルで修正とテストまで行う | PROC-CODE-LOCAL-EDIT |
| GitHub上でPR化したい | PROC-CODE-GH-PR |
| 調査結果を資料化したい | PROC-DOC-DRAFT |

## 注意点

Codex CLIはローカル状態を扱いやすい一方で、GitHub上のIssue、PR、CI結果は別途確認が必要になる場合があります。PRやCIまで含めて扱う場合はCodex Web/Appも候補にします。
