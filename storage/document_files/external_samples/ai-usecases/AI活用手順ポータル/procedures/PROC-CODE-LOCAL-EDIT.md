# PROC-CODE-LOCAL-EDIT Codex CLIで複数ファイルを修正する

| 項目 | 内容 |
|---|---|
| 手順ID | PROC-CODE-LOCAL-EDIT |
| ユースケース | 実装、複数ファイル修正 |
| パターン | ローカル複数ファイル修正 |
| AIツール | Codex CLI |
| 主な目的 | ローカル環境で既存コードを読み、複数ファイルを修正し、テストする |
| 主な成果物 | 差分、テスト結果、コミット候補、修正メモ |

## 向いている状況

- ローカルに対象リポジトリがある。
- 複数ファイルにまたがる変更が必要。
- テスト、lint、ビルドなどをローカルで実行したい。
- 未コミット差分やローカル設定も含めて作業したい。
- まず手元で安全に修正してから、別途PR化したい。

## 向いていない状況

- GitHub上でIssue起点にそのままPRを作りたい。
- ローカル環境がない。
- 調査だけで修正しない。

その場合は、`PROC-CODE-GH-PR` または `PROC-CODE-GH-READ` を検討します。

## 入力情報

| 入力 | 例 |
|---|---|
| ローカルパス | `/work/project-name` |
| 変更したい内容 | CSV importのバリデーションを追加する |
| 関連Issue / 要件 | Issue本文、要件メモ、エラー内容 |
| 実行したい検証 | `bundle exec rspec`、`npm test`、`rubocop` など |
| 制約 | 既存API互換、UI変更なし、特定ファイルは触らない等 |

## 手順

1. 対象リポジトリのローカルディレクトリでCodex CLIを起動する。
2. 現在のブランチ、未コミット差分、関連ファイルを確認させる。
3. 変更目的、制約、期待する成果物を伝える。
4. まず実装方針と変更予定ファイルを出させる。
5. 変更を小さな単位で適用させる。
6. 関連テスト、lint、ビルドなどを実行させる。
7. 失敗した場合はログを読ませ、原因を特定して修正させる。
8. 最後に差分、テスト結果、残課題をまとめさせる。

## 依頼文例

```text
このローカルリポジトリで、`〇〇機能` を修正してください。
まず関連ファイルと実装方針を整理し、その後に複数ファイルの修正を進めてください。
修正後は関連テストを実行し、差分、テスト結果、残課題をまとめてください。
```

## 判断ポイント

| 判断 | 対応 |
|---|---|
| 対象ファイルが多すぎる | 変更単位を分割する |
| テストが重い | まず関連テストだけ実行し、最後に広いテストを実行する |
| 仕様が曖昧 | ChatGPTで要件整理するか、確認事項を作る |
| GitHub上でPR化したい | Codex Web/Appまたは通常のgit操作でPR化する |

## 成果物テンプレート

```markdown
# 修正結果

## 対象

## 変更概要

## 変更ファイル

## 実行したテスト

## テスト結果

## 仕様・設計上の注意点

## 残課題

## PR化する場合の説明文案
```

## 注意点

ローカルでの複数ファイル修正は、実装と検証を進めやすい一方で、GitHub上のIssue、PR、CI状態とは分離されます。レビューやCI確認まで含める場合は、PR化後にCodex Web/AppまたはGitHub上のレビュー手順を使います。
